# MSGUI P3 실제 게임 화면 검증 — 2026-07-13

## 결론

공식 런처에서 간체중문을 선택한 뒤 P3 파일 전용 후보를 적용했고, 실제 게임의
SC 경로에서 한글이 정상 표시됐다. 메모리 패치, DLL 주입, 후킹, EXE 변경,
레지스트리 자동 변경은 사용하지 않았다.

- 메시지 후보: `B3203C89A99840AEC736936268803F685A7F2575E766AFB4C9D6E052383E86E6`
- 폰트 후보: `73E3759BF1886E95C769A95EB212F7ED34B7546E9A3DFA1EB49F542A7018E6B7`
- SC 경로 표지: `请点击鼠标`
- 오류 9001: 없음
- 정상 종료: 통과
- 원본 복원: 통과

## 확인한 화면

### 메인 메뉴

`새 게임`, `무장 편집`, `추가 콘텐츠`, `갤러리`, `설정`, `라이선스`,
`게임 종료`가 가로 방향 한글로 표시됐다. 공백, 대체 문자, 사각형 글리프,
명백한 버튼 잘림은 보이지 않았다.

- `reports/screenshots/msgui_p3_01_main_menu.png`
- `reports/screenshots/msgui_p3_01b_main_menu_lower.png`

표시됐다. 작은 능력치 레이블에서도 획 손실이나 세로쓰기 현상은 보이지 않았다.

## 판정 범위

저장된 화면에서 실제 확인한 ID만 화면 근거가 있는 것으로 기록했다.

`1, 5, 16, 17, 18, 19, 53, 56, 87, 88, 97, 105, 110, 178`

이번 실행으로 다음 항목을 확인했다.

- SC G1N 맵의 한글 Unicode 직접 조회: 통과
- Font-v3 대표 글리프 실제 렌더링: 통과
- 대표 메뉴의 가로쓰기와 버튼 수용: 통과
- 빈 한글 문구/대체 글리프/명백한 손상: 0건

다음은 통과로 주장하지 않는다.

- 설정 하위 화면
- 갤러리 하위 화면
- 번역 279개 전수
- 생성 한글 226자 전수
- 저장 화면에 나오지 않은 동적 printf 문구

## 종료 후 상태

게임은 자체 종료 확인창을 통해 정상 종료했다. 이후 두 설치 파일을 즉시 원본으로
복원했다.

- `MSG_PK/SC/msgui.bin`: `C2C69FDF09D9BE06E14F03C4F40562ADD0CA247EE0D50FC3E06EF501524B5E82`
- `RES_SC/res_lang.bin`: `916759185E9D64E487530DCA760CD36AE1FCFF021F39CEB1658837FE60AE0D99`
- 실행 중 게임 프로세스: 0

Font-v3와 메시지 P3의 런타임 게이트는 통과했다. 전체 공개 배포본의
`release_eligible`은 vNext 설치/복원/중단 복구와 독립 공개 안전 감사가 끝난 뒤
최종 판정한다.
