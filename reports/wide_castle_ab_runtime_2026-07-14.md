# 성 이름 wide-glyph A/B 실행 기록 — 2026-07-14

## 결론

처음 관찰한 설치 오류와 `ERROR:-9001`은 wide-glyph 후보의 실패가 아니었다.
게임 루트를 작업 디렉터리로 지정하지 않은 실행 때문에 발생했다. 동일한 순정 파일도
그 방식으로 실행하면 같은 오류가 났고, `F:\Games\NOBU16`을 작업 디렉터리로 지정한
직접 실행은 정상적으로 간체중문 메인 메뉴에 도달했다.

올바른 실행 조건에서 A/B 결과는 다음과 같다.

| 조합 | 결과 |
|---|---|
| 순정 메시지 + 순정 폰트 | 간체중문 메인 메뉴 정상 진입 |
| 시험 메시지 + 순정 폰트 | 간체중문 메인 메뉴 정상 진입 |
| 순정 메시지 + 시험 폰트 | 간체중문 메인 메뉴 정상 진입 |
| 시험 메시지 + 시험 폰트 | 간체중문 메인 메뉴와 새 게임 세력 선택 화면 정상 진입 |

따라서 메시지 wrapper, 프록시 문자 치환, wide G1N 레코드, 두 파일의 결합은 모두
부팅을 통과했다. 이전의 “큰 glyph metric 때문에 시작 실패” 가설은 실제 A/B 결과로
기각됐다.

## 시각 판정 보류

결합 후보로 실제 전략 지도에 들어가기 직전, 사용자가 성 이름 세로 표시를 허용하고
번역을 우선하라고 결정했다. 이에 따라 wide bitmap의 실제 지도 방향·잘림·공용 `城`
접미사 배치는 판정하지 않고 실험을 중단했다. 이 방식은 공개 패치의 필수 구성에서
제외하고 연구용 workstream으로만 보존한다.
