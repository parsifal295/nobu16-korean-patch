# `ERROR:-9001` 실행 조건 확인

날짜: 2026-07-14

## 결론

`ERROR:-9001`은 이 환경에서 수정 리소스의 해시 때문이 아니라 게임 프로세스의 시작 위치(작업 폴더)가 게임 설치 폴더가 아닐 때 재현됐다.

## 재현

- SC 원본/수정 리소스 조합과 EN 원본 리소스에서 EXE를 작업 폴더 지정 없이 직접 실행하면 동일한 `ERROR:-9001`이 발생했다.
- SC 원본 리소스로 복구한 뒤 게임 설치 폴더를 작업 폴더로 지정해 `NOBU16PK.exe`를 실행하면 `ERROR:-9001` 없이 전체 게임 창이 생성됐다.
- 따라서 `ERROR:-9001`만으로 메시지 또는 폰트 리소스 변조 거부를 판정할 수 없다.

## 프로젝트 규칙

- 검증 대상은 Steam에서 받은 설치본이 아닌 비 Steam PC 설치본이다. Steam 실행 옵션이나 Steam 설치 경로를 전제로 진단하지 않는다.
- 런타임 QA는 항상 게임 설치 폴더를 프로세스 작업 폴더로 지정한다.
- 배포 설치기는 게임을 자동 실행하지 않는다.
- 사용자 안내에는 바로가기의 `시작 위치`를 게임 설치 폴더로 설정해야 한다고 명시한다.
- `ERROR:-9001`이 발생하면 파일 해시 조사 전에 작업 폴더 조건부터 확인한다.
