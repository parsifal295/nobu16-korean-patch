# 번역 경계 검증 보강 (2026-07-14)

## 목적

현재 공개 오버레이에는 문제가 없었지만, 이후 번역 배치가 잘못 작성됐을 때 검증을 우회하거나 이전 번역이 남을 수 있는 경계를 독립 검토 후 보강했다.

## 교정 사항

- 출처 언어가 없는 `line_breaks` 예외를 제거했다. 모든 불변식 예외는 `printf:JP`, `line_breaks:JP`처럼 실제 공식 언어 문자열을 검증 기준으로 지정해야 한다.
- `invariant_overrides`를 쓰는 공개 번역은 반드시 `reviewed` 상태여야 한다.
- 파일명 순서상 뒤에 오는 배치가 공백 번역으로 같은 ID를 철회하면 앞선 병합 결과를 제거한다.
- `str.strip()`이 제거하지 못하는 U+200B ZERO WIDTH SPACE, U+FEFF, 단독 결합 부호도 의미 있는 번역으로 인정하지 않는다. 게임의 PUA 아이콘은 계속 의미 있는 표시 토큰으로 인정한다.

## 검증

- 단위 테스트 14개 통과.
- 공개 오버레이 3,951개를 실제 개발 카탈로그와 번역 배치에서 다시 내보냈다.
- 재생성 파일은 추적 중인 공개 오버레이와 byte-exact이며 SHA-256은 `FDB325F3729E8853705F77A54ED248AFAA3C9E247457664749082A13A57BB477`이다.
- 현재 공개 오버레이의 source-qualified override 8개는 모두 `reviewed`이며 Unicode Format 문자는 0개다.

이 변경은 검증 코드만 강화하며 v0.3-dev 메시지·폰트 target 해시에는 영향을 주지 않는다.
