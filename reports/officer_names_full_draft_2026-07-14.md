# 장수명 전체 초안 v0.1 빌드 보고서

## 범위

- 대상 리소스: `MSG_PK/SC/msgev.bin`
- 실제 장수 슬롯: ID `0..2206`, 총 2,207명
- 제외 슬롯: ID `2207..2399`의 언어별 dummy 193개
- 부족 두목, 일반 병사·전령, 이벤트 화자 및 `[b...]` 참조는 장수명 범위에서 제외

초기 조사에서 얻은 2,573개는 장수 외 문자열까지 포함한 수치였다. SC/JP/EN 테이블을
다시 정렬한 결과 실제 장수 슬롯은 0부터 2206까지 연속된 2,207개임을 확인했다.

## 표기 생성

- 공식 SC/JP/EN 테이블을 같은 ID로 정렬했다.
- EN의 `ª`, `¥`, `¨`를 각각 `ō`, `Ō`, `ū`로 복원해 독음으로 사용했다.
- 일본어 후리가나 목록은 비공개 QA 입력으로만 캐시했고 Git이나 배포본에 포함하지 않았다.
- 후리가나와 공식 EN 독음이 일치한 1,866명은 교차 확인 결과를 사용했다.
- 후리가나가 공식 EN과 어긋난 298명은 게임의 공식 EN 독음을 우선했다.
- 표시된 JP 성과 EN의 대체 성이 다른 명확한 1건(明智光安/Dota)은 JP 표시에 맞췄다.
- 영문 순서가 앞뒤로 바뀐 복합명 3건과 칭호·단일명 37건은 별도 규칙으로 처리했다.
- `小松 / Ko Matsu`는 한 단어 `고마츠`로 수동 고정했다.

프로젝트 표기는 성과 이름 사이 ASCII 공백 하나를 둔다. 사용자 예시에 맞춰 `つ`는
`츠`로 적고, `오다 노부나가`, `이시다 미츠나리`, `아츠지 사다유키`를 회귀 기준으로
고정했다. `ō`와 `ū`는 별도 음절로 늘이지 않되 `한베에`처럼 실제 표기에 필요한 `ええ`
계열은 유지한다.

## 배포 경계

비공개 카탈로그는 `data/translations/`와 `tmp/`에만 존재한다. 공개 오버레이에는 다음만
들어 있다.

- 숫자 ID
- SC 원문의 UTF-16LE SHA-256
- 프로젝트가 작성한 한국어 이름
- 상태와 명시적 안전 플래그

공개 파일 `data/public/msgev_ko_officer_names_0000_2399.v0.1.json`에서 CJK 통합한자 수는
0개였고, JP/SC/EN 원문 문자열은 포함되지 않았다.

## 남은 검수

이 단계는 전체 이름의 **음역 초안**이다. 희귀 인물의 관용 표기와 `카/가`, `타/다`,
`주/쥬` 같은 팬덤 표기 차이는 별도 교정 목록으로 계속 다듬는다. 전체본을 게임에 넣기
전에 다음이 필요하다.

1. 장수명 전체 글자를 포함하는 재배포 가능 폰트 v5 생성
2. `msgdata.bin`의 성·이름 분리 슬롯 동기화
3. 다중 리소스 설치·복원 거래 추가
4. 장수 목록, 편집, 상세 화면의 길이·공백·잘림 런타임 검수
