# Switch v1.3 기반 PK `msgev` 구조 복구 248건

이 작업 스트림은 Switch v1.3의 텍스트가 v1.1과 바이트 단위로 같다는 입력 핀을 사용해, 엄격 전송에서 남은 261건을 다시 검사한다. 문장 조각이나 플레이스홀더를 재배치하지 않고 자동으로 증명할 수 있는 248건만 `MSG_PK/SC/msgev.bin`용 소스 프리 overlay로 만들었다.

## 결과

- 잔여 후보: 261건
- 자동 복구: 248건
- 보류: 13건
- 선택 ID SHA-256: `C3B4DD558398729877C9861CF917B1C4BE2567B87CE460763FA68C7A529459D0`
- overlay SHA-256: `D2757409D426148CE2E636828F4693830D66A661236C4F6E816DD0D028D48F1C`
- 정확한 PK 번역 대상 12,906건과의 교집합: 248건
- 기존 소유 overlay 및 선행 245건과의 교집합: 0건

적용한 기계적 복구는 다음 네 종류다. 여러 종류가 한 항목에 함께 적용될 수 있다.

- 줄바꿈 템플릿: 179건. 기존 한국어 공백 경계만 사용해 SC의 줄바꿈 개수와 형식을 맞췄다.
- ESC 토큰 템플릿: 70건. 양쪽 토큰 개수가 같을 때만 기존 위치를 유지한 채 서수별로 SC 토큰을 붙였다.
- 끝 공백 템플릿: 6건. Switch 값에 끝 공백이 없고 SC 계약에만 있을 때 그대로 붙였다.
- U+30FB 가운데점 정규화: 4건. U+00B7로 바꿔 소스 스크립트를 제거했다.

모든 선택 항목은 ESC와 공백을 형식 중립화한 한국어 가시 문자열 및 단어 경계 순서가 전후 동일하다. printf, 알 수 없는 `%`, 앞뒤 공백, ESC 순서, 제어문자, 줄바꿈, PUA, 사용자 정의 대괄호 토큰도 최종 SC 계약과 정확히 같다.
