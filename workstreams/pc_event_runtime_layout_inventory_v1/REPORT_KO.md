# PK 이벤트 런타임 토큰 레이아웃 전수 감사

## 결론

W101 strict 한국어 후보의 런타임 행 **1,049행**을 Static Patch 007 기준으로 전수 계측했습니다.
인명 숫자 토큰 행 **859행**은 모두 최대 4줄·각 줄 유효폭 912px 이하로 **통과**했습니다.
폭 또는 줄 수 초과 후보는 **0행**입니다.
`%s`/`%d`처럼 실제 런타임 값의 행별 상한을 입증할 수 없는 **190행**은 안전하게 hold로 남겼습니다. 이 hold는 레이아웃 통과 판정이 아니라, 값 상한 증거가 없다는 뜻입니다.

## 기준과 범위

- PK `MSGEV`만 적용: 30px, 줄 간격 8, 유효폭 912px, 최대 4줄.
- 원본 G1N 기준 폭은 전각 48px/반각 24px이며, 유효폭은 `ceil(raw_g1n_width_px * 30 / 48)`입니다.
- 숫자 인명 토큰은 같은 ID의 strict 한국어 이름 전체 폭을 예약했습니다. prefix의 런타임 의미는 추론하지 않았고 모든 예약은 `runtime_proven=false`입니다.
- JP/EN/SC/TC는 의미·태그·토큰 구조 확인용 읽기 전용 증거입니다. 타 언어의 줄바꿈을 한국어에 이식하지 않았습니다.

## W100 → W101 재베이스

W101의 변경 ID는 3489, 3490, 3491, 3493, 3497, 3500, 3502, 3505, 3506, 3508, 3510, 3514, 3516, 3522, 3526입니다. 이 중 런타임 행은 3514, 3522, 3526이며, 해당 3행도 W101 새 문안 기준으로 다시 계측했습니다. W100과 W101의 런타임 행 범위는 동일한 1,049행입니다.

행별 표시 문자열(확정 불가 hold는 `null`), raw/effective 폭, 전각/반각 수, 줄 수, 912px 초과 여부, 토큰 예약·보류 사유는 `public/pc_event_runtime_layout_inventory.v1.json`에 전부 기록했습니다.

검증 상태: **PASS**. Steam 파일·후보 바이너리·Git·네트워크·릴리스에는 쓰지 않았습니다.
