# P3 실제 화면 QA 매트릭스

이 문서는 공식 런처가 간체중문(`简体中文`)으로 설정된 상태에서만 사용한다.
현재 P3는 `development_milestone=true`, `release_eligible=false`이며, 아래 화면을
실제로 확인하기 전에는 그 값을 올리지 않는다.

## 공통 실행 조건

- 게임 프로세스와 공식 런처가 닫힌 상태에서 P3를 적용한다.
- 적용 직후 두 대상 SHA-256을 고정값과 비교한다.
  - `MSG_PK/SC/msgui.bin`: `B3203C89A99840AEC736936268803F685A7F2575E766AFB4C9D6E052383E86E6`
  - `RES_SC/res_lang.bin`: `73E3759BF1886E95C769A95EB212F7ED34B7546E9A3DFA1EB49F542A7018E6B7`
- `NOBU16PK.exe`의 작업 디렉터리는 게임 루트로 지정한다. 그렇지 않으면 오류
  9001이 발생할 수 있다.
- 메모리 접근, DLL 주입, 후킹, EXE 변경, 레지스트리 자동 변경은 사용하지 않는다.
- 각 화면에서 한글 표시 여부뿐 아니라 가로쓰기, 빈 글리프, 대체 글자, 겹침,
  잘림, 줄바꿈, 버튼 폭을 함께 기록한다.

## 판정 규칙

- `PASS`: 화면에 실제 표시된 해당 ID만 `surface_evidence`와 함께 기록한다.
- `FAIL_FONT`: 사각형/공백/다른 글자/획 손실이 보인다.
- `FAIL_LAYOUT`: 문구 잘림, 겹침, 비정상 세로쓰기, 버튼 영역 초과가 보인다.
- `FAIL_TEXT`: 문맥상 오역이나 용어 불일치가 보인다.
- `NOT_VISIBLE`: 번역은 포함됐지만 이번 화면에 표시되지 않았다. 통과로 세지 않는다.
- 동적 `%` 문자열은 실제 값을 넣은 화면을 별도로 확인하기 전에는 토큰 단위 검사만
  통과한 것으로 기록한다.
