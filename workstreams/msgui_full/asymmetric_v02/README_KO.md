# MSGUI 비대칭 슬롯 v0.2 번역 후보

공식 SC 가로쓰기 경로의 일반 비공백 UI는 v0.1에서 이미 모두 처리됐다. 이 후보는
SC·TC 원문은 공백이지만 EN 또는 JP에만 의미가 남은 108개 슬롯을 다시 분류한다.

- 언어 중립 또는 내부 효과명 86개: 한국어 초벌 적용
- 일본어 읽기·IME 전용 22개: SC 레이아웃과 기능을 바꾸지 않도록 공백 보존
- 전 언어 구조 공백 3개(ID 733, 734, 1607): canonical empty

22개 보존 ID는 `513, 689, 691, 1302, 1350, 1352, 1694, 1921–1932,
2570–2572`다. 이 항목에 한국어를 넣으면 SC에는 없던 읽기 입력칸이나 일본어 IME
축약 버튼을 활성화할 수 있으므로 번역 누락으로 세지 않는다.

86개 번역 범위는 `2329–2348, 2408–2409, 2419–2420, 2457, 2459, 2558,
2650, 2657, 2661, 2691–2746`이다. ID 2657은 SC가 공백인 printf 문장이라 공식 EN의
토큰 순서를 근거로 `printf:EN` 검수 override를 사용한다. ID 2729·2730은 EN의 병력
회복과 JP의 체력 회복이 충돌하여 효과 배열 구조와 EN을 우선했으며 런타임 확인 대상으로
남긴다.

Font-v5가 사용하는 MSGUI 수요 파일과 바이트 단위로 동일하며, 이 UI 확장만을 위해 폰트를
다시 만들 필요는 없다.

## 검증 결과와 승격 상태

- private 개발 카탈로그: `empty=1,041`, `untranslated=22`, `translated=4,019`,
  `reviewed=18`, 오류·경고 0
- 남은 22개는 위에서 명시한 일본어 전용 공백 보존 항목뿐이다.
- overlay export, public replay, 메시지 build, recipe export를 서로 다른 두 경로에서
  수행했고 각 산출물이 byte-exact였다.
- recipe를 순정 fixture에 다시 적용해 target 해시를 재현했다.
- 설치된 게임 파일은 수정하지 않았다.

이 후보는 전체 UI 번역 범위를 논리적으로 닫지만 86개가 원래 SC 공백이던 내부 슬롯이므로
화면·전투 효과 검수 전에는 배포 승격하지 않는다. `release_eligible=false`이며, 다음 공개본은
기존 2파일 P4 설치기가 아니라 장수명 RC의 4리소스 거래·복구 구조를 일반화해 만든다.

## 재현

개발용 다국어 batch는 상용 원문을 포함하므로 Git에서 제외한다. 추적되는
`tools/fill_msgui_asymmetric_catchup.py`에는 숫자 ID와 프로젝트 소유 한국어만 들어 있다.
공개 overlay와 recipe에는 숫자 ID, SC 원문 해시, 한국어 교체문, 구조 메타데이터만 있으며
완성 `msgui.bin`이나 상용 원문은 없다.

모든 재현은 설치본 대신 고정 순정 fixture에서 수행한다. 설치본은 현재 장수명 RC가 적용된
상태일 수 있으므로 stock 입력으로 사용하면 안 된다.
