# msggame.bin 번역 배치 3 v0.3

MSG_PK/SC/msggame.bin 좌표 기반 한국어 초벌 번역의 세 번째 배치다. 상용 원문과
완성 바이너리는 저장하지 않으며, 공개 오버레이에는 좌표·SC UTF-16LE SHA-256·
프로젝트 한국어만 들어간다.

## 범위와 상태

| 항목 | 값 |
|---|---:|
| 번역 리터럴 | 150개 |
| 대조 레코드 | 115개 |
| 스캔한 표시 후보 | 150개 |
| 비언어 형식 조각 보류 | 0개 |
| 첫 좌표 | (2, 298, 0) |
| 마지막 좌표 | (2, 565, 0) |
| 다음 연속 좌표 | (2, 566, 0) |
| 런타임 검수 | 미실시 |

- 블록 2 레코드 298~565를 순서대로 대조하고 언어 의미가 있는 다음 표시 리터럴
  150개를 모두 번역했다. 빈 문자열과 추가 특성용 더미처럼 화면에 표시되지 않는
  레코드는 범위에서 제외했다.
- 이 범위에는 별도로 보류할 내부 키·코드·비언어 표시 조각이 없었다. 선택 좌표가
  실제 SC 표시 후보의 연속된 다음 150개인지 회귀 테스트로 다시 계산한다.
- 언어별 리터럴 분할 수가 다르므로 JP·EN·TC의 같은 literal_id를 같은 문장으로
  간주하지 않았다. 같은 block_id·record_id 레코드 전체를 수동 대조한 뒤 SC
  좌표에만 번역을 연결했다.
- 동적 이름과 수치가 끼어드는 문장은 현 구조에서 앞뒤 공백을 바꿀 수 없으므로
  콜론과 조사를 이용해 결합했다. 전 항목은 게임 화면 검수가 필요한 초벌이다.

## 산출물

- build_translation_batch3.py: 고정 설치본을 읽어 공개 산출물을 결정적으로
  생성하고 오프라인 msggame.bin을 메모리에서 재패킹·재파싱한다.
- public/msggame_ko_system_messages_b02r0298_0565.v0.3.json: 공용 리터럴
  빌더와 호환되는 source-free 번역 오버레이.
- evidence/translation_alignment_evidence.v0.3.json: 언어별 같은 레코드의
  구조와 원문 해시만 기록한 정렬 근거.
- review/translation_review_index.v0.3.json: 좌표별 초벌·사람 검수·런타임 검수
  상태와 주의점을 기록한 검수 목록.
- translation_validation.v0.3.json: 재현성, 바이너리 구조, 문자열 불변 조건,
  설치본 불변과 배포 안전성 검사 결과.

## 재생성

~~~powershell
python -B workstreams/msggame/build_translation_batch3.py `
  --out-root workstreams/msggame
python -B -m unittest workstreams.msggame.tests.test_translation_batch3 -v
~~~

생성기는 설치된 SC·JP·EN·TC 파일의 크기와 packed/raw SHA-256을 먼저 확인한다.
고정 설치본과 다르면 쓰기 전에 중단하며 설치 경로에는 어떤 파일도 쓰지 않는다.

## 검증한 항목

- 오버레이 A/B/final 생성 결과의 바이트 일치
- 오프라인 재패킹 바이너리 A/B/final SHA-256 일치
- 18개 최상위 블록, 21,581개 레코드와 모든 리터럴 좌표 유지
- 선택한 150개 좌표만 교체하고 다음 시작 좌표 (2, 566, 0) 고정
- 가변 길이 한국어 적용 뒤 블록·레코드 오프셋 재계산과 4바이트 정렬 확인
- 리터럴 밖 불투명 바이트코드의 바이트 일치
- 줄바꿈, 앞뒤 공백, printf 토큰, PUA 코드포인트와 괄호·인용부호 순서 유지
- 공개 JSON의 한자 통합영역·가나 원문 노출 0
- 설치된 SC·JP·EN·TC 파일의 작업 전후 SHA-256 일치

## 남은 검수

동적 이름·정책명·수치가 리터럴 사이에 삽입되는 문장은 실제 화면에서 조사와
콜론 결합을 확인해야 한다. 공개 오버레이의 150개 항목은 모두 사람 검수 필요,
런타임 검수 0개로 표시되어 있다.
