# msggame.bin 번역 배치 1 v0.1

`MSG_PK/SC/msggame.bin`의 좌표 기반 첫 한국어 초벌 번역 배치다. 상용 원문과
완성 바이너리는 저장하지 않으며, 공개 오버레이에는 좌표·SC UTF-16LE SHA-256·
프로젝트 한국어만 들어간다.

## 범위와 상태

| 항목 | 값 |
|---|---:|
| 번역 리터럴 | 150개 |
| 완결 레코드 | 91개 |
| 첫 좌표 | `(1, 3, 0)` |
| 마지막 좌표 | `(2, 197, 1)` |
| 다음 연속 좌표 | `(2, 198, 0)` |
| 런타임 검수 | 미실시 |

- 블록 1 레코드 3의 표시 문장 1개와 블록 2 레코드 86~197의 비어 있지 않은
  표시 리터럴 149개를 번역했다.
- 블록 0의 표시 후보 93개는 대명사·호칭·조사와 결합되는 문법 조각이다. 화면
  문맥 없이 바꾸면 문장 전체가 어긋날 위험이 커서 제외한 것이 아니라 후속
  문맥 전용 배치로 보류했다.
- 이번 연속 범위에서 내부 키나 코드 문자열로 판정해 제외한 항목은 없다.
- 언어별 리터럴 분할 수가 다르므로 JP·EN·TC의 같은 `literal_id`를 같은 문장으로
  간주하지 않았다. 같은 `(block_id, record_id)` 레코드를 수동 대조한 뒤 SC 좌표에만
  번역을 연결했다.

## 산출물

- `build_translation_batch1.py`: 고정 설치본을 읽어 아래 공개 산출물을 결정적으로
  생성하고 오프라인 `msggame.bin`을 메모리에서 재빌드·재파싱한다.
- `public/msggame_ko_system_messages_b01r0003_b02r0086_0197.v0.1.json`:
  `build_literal_overlay.py`와 호환되는 source-free 번역 오버레이다.
- `evidence/translation_alignment_evidence.v0.1.json`: 언어별 같은 레코드의
  구조와 원문 해시만 기록한 정렬 근거다.
- `review/translation_review_index.v0.1.json`: 각 좌표의 초벌·인간 검수·런타임
  검수 상태와 주의점을 기록한다.
- `translation_validation.v0.1.json`: 재현성, 바이너리 구조, 문자열 불변 조건,
  설치본 불변과 배포 안전성 검증 결과다.

## 재생성

```powershell
python workstreams/msggame/build_translation_batch1.py `
  --out-root workstreams/msggame
python -m unittest workstreams.msggame.tests.test_translation_batch1 -v
```

생성기는 설치된 SC·JP·EN·TC 파일의 크기와 packed/raw SHA-256을 먼저 확인한다.
지원 대상과 다른 파일이면 쓰기 전에 중단하며, 설치본 경로에는 아무 파일도 쓰지
않는다.

## 검증된 항목

- 오버레이 A/B/final 생성 결과 바이트 동일
- 오프라인 재빌드 바이너리 A/B/final SHA-256 동일
- 18개 상위 블록과 21,581개 레코드 좌표 유지
- 25,598개 리터럴 좌표 유지 및 선택 좌표 150개만 교체
- 가변 길이 번역 후 상위 블록·레코드 오프셋 재계산과 4바이트 정렬 확인
- 리터럴 밖 불투명 바이트코드 동일
- 제어문자, 개행 순서, printf 토큰, PUA 코드포인트, 괄호·인용부호 순서 유지
- 공개 JSON의 한자 통합영역·가나 원문 누출 0
- 설치된 SC·JP·EN·TC 파일의 작업 전후 SHA-256 동일

## 런타임 주의점

이 배치는 파일 구조와 오프라인 재빌드까지 검증했지만 게임 화면 검수는 아직 하지
않았다. 특히 동적 이름·세력·수치가 리터럴 사이에 삽입되는 레코드는 한국어 조사,
콜론 접합, 줄 너비를 실제 화면에서 확인해야 한다. 공개 오버레이는 초벌 번역으로
표시되며 인간 검수 150개와 런타임 검수 150개가 모두 남아 있다.
