# msggame.bin 번역 배치 6 v0.6

MSG_PK/SC/msggame.bin 좌표 기반 한국어 초벌 번역의 여섯 번째 배치다. 상용 원문과
완성 바이너리는 저장하지 않으며, 공개 오버레이에는 좌표·SC UTF-16LE SHA-256·
프로젝트 한국어만 들어간다.

## 범위와 상태

| 항목 | 값 |
|---|---:|
| 번역 리터럴 | 150개 |
| 누적 번역 리터럴 | 900개 |
| 대조 레코드 | 148개 |
| 스캔한 표시 후보 | 150개 |
| 코드·비언어 조각 보류 | 0개 |
| 첫 좌표 | (4, 76, 0) |
| 마지막 좌표 | (6, 559, 0) |
| 다음 연속 좌표 | (6, 560, 0) |
| 런타임 검수 | 미실시 |

- 블록 4 레코드 76부터 블록 6 레코드 559까지 좌표 순으로 확인하고 다음 언어 의미
  리터럴 150개를 모두 번역했다. 이번 범위에는 코드·비언어 표시 후보와 비표시
  리터럴이 없다.
- 외교·편집 UI, 환경 설정 도움말, 파워업키트 안내와 평정 반응 대사를 포함한다.
- SC·JP·EN·TC의 같은 block_id·record_id 레코드 전체를 대조했다. 언어마다
  리터럴 분할 수가 달라 같은 literal_id를 같은 문장이라고 가정하지 않았다.
- 동적 세력명과 취락명이 들어가는 레코드 2개는 콜론과 화살표로 어순을 재구성했다.
  실제 값이 결합된 화면에서 별도 확인해야 한다.

## 산출물

- build_translation_batch6.py: 고정 설치본을 읽어 공개 산출물을 결정적으로
  생성하고 오프라인 msggame.bin을 메모리에서 재패킹·재파싱한다.
- public/msggame_ko_system_messages_b04r0076_b06r0559.v0.6.json: 공용 리터럴
  빌더와 호환되는 source-free 번역 오버레이.
- evidence/translation_alignment_evidence.v0.6.json: 언어별 같은 레코드의
  구조와 원문 해시만 기록한 정렬 근거.
- review/translation_review_index.v0.6.json: 좌표별 초벌·사람 검수·런타임 검수
  상태와 동적 결합 주의점을 기록한 검수 목록.
- translation_validation.v0.6.json: 재현성, 바이너리 구조, 문자열 불변 조건,
  설치본 불변과 배포 안전성 검사 결과.

## 재생성

~~~powershell
python -B workstreams/msggame/build_translation_batch6.py `
  --out-root workstreams/msggame
python -B -m unittest workstreams.msggame.tests.test_translation_batch6 -v
~~~

생성기는 설치된 SC·JP·EN·TC 파일의 크기와 packed/raw SHA-256을 먼저 확인한다.
고정 설치본과 다르면 쓰기 전에 중단하며 설치 경로에는 어떤 파일도 쓰지 않는다.

## 검증한 항목

- 오버레이 A/B/final 생성 결과의 바이트 일치
- 오프라인 재패킹 바이너리 A/B/final SHA-256 일치
- 18개 최상위 블록, 21,581개 레코드와 모든 리터럴 좌표 유지
- 다음 표시 후보 150개 전부 번역, 임의 누락과 코드 조각 혼입 없음
- 이전 v0.1~v0.5 오버레이 750개 좌표와 중복 없음
- 가변 길이 한국어 적용 뒤 블록·레코드 오프셋 재계산과 4바이트 정렬 확인
- 리터럴 밖 불투명 바이트코드의 바이트 일치
- 줄바꿈, 앞뒤 공백, printf 토큰, PUA 코드포인트와 괄호·인용부호 순서 유지
- 공개 JSON의 한자 통합영역·가나 원문 노출 0
- 설치된 SC·JP·EN·TC 파일의 작업 전후 SHA-256 일치

## 남은 검수

동적 세력명·취락명의 결합 어순과 긴 환경 설정·상품 안내의 줄바꿈을 실제 화면에서
확인해야 한다. 평정 반응 대사는 화자 성격과 말투에 따라 다듬을 수 있다. 공개
오버레이의 150개 항목은 모두 사람 검수 필요, 런타임 검수 0개로 표시되어 있다.
