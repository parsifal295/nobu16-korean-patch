# msggame.bin 번역 배치 20 v0.20

`MSG_PK/SC/msggame.bin` 좌표 기반 한국어 초벌 번역의 다음 연속 배치다.
공개 산출물에는 원문이나 완성 게임 리소스를 넣지 않고, 좌표·SC UTF-16LE
SHA-256·한국어 번역문만 기록한다.

## 범위와 상태

| 항목 | 값 |
|---|---:|
| 이번 번역 리터럴 | 150개 |
| 참조 레코드 | 85개 |
| 기준 언어 | SC·JP·EN·TC |
| 첫 좌표 | `(6, 3478, 0)` |
| 마지막 좌표 | `(6, 3697, 1)` |
| 다음 연속 좌표 | `(6, 3698, 0)` |
| 런타임 검수 | 미실시 |

- 공훈·논공행상·승진 대사, 혼인·보좌 대사, 가보 수여·교환·박탈,
  관직 추천과 동맹 만료 알림을 포함한다.
- SC·JP·EN·TC의 같은 블록·레코드 전체 문맥을 대조한다. 언어마다 리터럴
  분할 수가 달라질 수 있으므로 다른 언어의 `literal_id`를 직접 대응하지
  않는다.
- v0.19에서만 필요했던 빈 리터럴 스캔 확장은 이 배치에 적용하지 않는다.
  원래의 보이는 SC 리터럴 판정으로 다음 자연 경계 150개를 다시 고정했다.
- 동적 인물·가문·관직·가보 결합, 각 슬롯의 줄바꿈과 공백은 실제 화면에서
  재검수해야 한다.

## 새 산출물

1. `build_translation_batch20.py`: 고정된 SC·JP·EN·TC 설치본을 읽기 전용으로
   검증하고 공개 오버레이와 오프라인 재패킹 검증 결과를 결정적으로 생성한다.
2. `public/msggame_ko_system_messages_b06r3478_3697.v0.20.json`: 원문 없는
   좌표 기반 번역 오버레이.
3. `evidence/translation_alignment_evidence.v0.20.json`: 원문 없이 기록한
   언어별 같은 레코드 정렬 근거.
4. `review/translation_review_index.v0.20.json`: 좌표별 사람·화면 검수 상태.
5. `translation_validation.v0.20.json`: 재현성·바이너리 구조·안전성 검증 결과.
6. `tests/test_translation_batch20.py`: 범위·v0.19 격리·원문 부재·오버레이 충돌·
   A/B 결정성·재패킹 회귀 테스트.

## 재생성 및 좁은 검증

```powershell
python -B workstreams/msggame/build_translation_batch20.py `
  --out-root workstreams/msggame
python -B -m unittest workstreams.msggame.tests.test_translation_batch20 -v
```

생성기는 설치된 SC·JP·EN·TC 파일의 packed/raw 크기와 SHA-256을 먼저
검사한다. 설치 파일, 기본판 파일, 폰트, 설치기, 실행 파일, 메모리,
레지스트리는 변경하지 않는다.
