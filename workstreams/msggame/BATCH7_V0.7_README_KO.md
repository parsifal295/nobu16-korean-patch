# msggame.bin 번역 배치 7 v0.7

MSG_PK/SC/msggame.bin 좌표 기반 한국어 초벌 번역의 일곱 번째 배치다. 상용 원문과
완성 바이너리는 저장하지 않으며, 공개 오버레이에는 좌표·SC UTF-16LE SHA-256·
프로젝트 한국어만 들어간다.

## 범위와 상태

| 항목 | 값 |
|---|---:|
| 번역 리터럴 | 150개 |
| 누적 번역 리터럴 | 1,050개 |
| 대조 레코드 | 132개 |
| 스캔한 표시 후보 | 150개 |
| 코드·비언어 조각 보류 | 0개 |
| 첫 좌표 | (6, 560, 0) |
| 마지막 좌표 | (6, 947, 0) |
| 다음 연속 좌표 | (6, 948, 0) |
| 런타임 검수 | 미실시 |

- 블록 6 레코드 560부터 947까지 좌표 순으로 확인하고 다음 언어 의미 리터럴
  150개를 모두 번역했다. 이번 범위에는 코드·비언어 표시 후보와 비표시 리터럴이
  없다.
- 미확인 헌책·구신 반응, 공략 부대와 적대 세력 반응, 지행지 관리, 격무와 평정중
  합류, 가신 불만, 부상·질병, 정책 및 군량 관련 평정 대사를 포함한다.
- SC·JP·EN·TC의 같은 block_id·record_id 레코드 전체를 대조했다. 언어마다
  리터럴 분할 수가 달라 같은 literal_id를 같은 문장이라고 가정하지 않았다.
- 동적 무장명·부대명이 끼는 18개 복수 리터럴 레코드는 SC의 삽입 순서를 유지하며
  콜론과 조사를 배치했다. 실제 값이 결합된 화면에서 별도 확인해야 한다.

  생성하고 오프라인 msggame.bin을 메모리에서 재패킹·재파싱한다.

- translation_validation.v0.7.json: 재현성, 바이너리 구조, 문자열 불변 조건,
  설치본 불변과 배포 안전성 검사 결과.

## 재생성

~~~powershell
python -B workstreams/msggame/build_translation_batch7.py `
  --out-root workstreams/msggame
python -B -m unittest workstreams.msggame.tests.test_translation_batch7 -v
~~~

생성기는 설치된 SC·JP·EN·TC 파일의 크기와 packed/raw SHA-256을 먼저 확인한다.
고정 설치본과 다르면 쓰기 전에 중단하며 설치 경로에는 어떤 파일도 쓰지 않는다.

## 검증한 항목

- 오버레이 A/B/final 생성 결과의 바이트 일치
- 오프라인 재패킹 바이너리 A/B/final SHA-256 일치
- 18개 최상위 블록, 21,581개 레코드와 모든 리터럴 좌표 유지
- 다음 표시 후보 150개 전부 번역, 임의 누락과 코드 조각 혼입 없음
- 이전 v0.1~v0.6 오버레이 900개 좌표와 중복 없음
- 가변 길이 한국어 적용 뒤 블록·레코드 오프셋 재계산과 4바이트 정렬 확인
- 리터럴 밖 불투명 바이트코드의 바이트 일치
- 줄바꿈, 앞뒤 공백, printf 토큰, PUA 코드포인트와 괄호·인용부호 순서 유지
- 공개 JSON의 한자 통합영역·가나 원문 노출 0
- 설치된 SC·JP·EN·TC 파일의 작업 전후 SHA-256 일치

## 남은 검수

동적 무장명·부대명과 한국어 조각의 결합 어순을 실제 화면에서 확인해야 한다.
평정 반응 대사는 화자 성격과 말투에 따라 다듬을 수 있다. 공개 오버레이의 150개
항목은 모두 사람 검수 필요, 런타임 검수 0개로 표시되어 있다.
