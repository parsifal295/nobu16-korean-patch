# msggame.bin 번역 배치 5 v0.5

MSG_PK/SC/msggame.bin 좌표 기반 한국어 초벌 번역의 다섯 번째 배치다. 상용 원문과
완성 바이너리는 저장하지 않으며, 공개 오버레이에는 좌표·SC UTF-16LE SHA-256·
프로젝트 한국어만 들어간다.

## 범위와 상태

| 항목 | 값 |
|---|---:|
| 번역 리터럴 | 150개 |
| 누적 번역 리터럴 | 750개 |
| 대조 레코드 | 134개 |
| 스캔한 표시 후보 | 153개 |
| 비언어 형식 조각 보류 | 3개 |
| 비표시 공백 원형 유지 | 2개 |
| 첫 좌표 | (2, 666, 0) |
| 마지막 좌표 | (4, 75, 0) |
| 다음 연속 좌표 | (4, 76, 0) |
| 런타임 검수 | 미실시 |

- 블록 2 레코드 666부터 블록 4 레코드 75까지 좌표 순으로 확인하고 다음 언어 의미
  리터럴 150개를 번역했다. 문자열 슬롯이 없는 레코드와 블록은 자연스럽게
  건너뛰었다.
- 좌표 (2, 676, 2)의 수치 앞 기호와 (4, 57, 1)·(4, 57, 2)의 반복 글머리표는
  비언어 형식 조각이다. 번역 항목으로 세지 않고 원본 바이트를 유지했으며, 좌표와
  보류 사유를 공개 근거 및 검증 파일에 기록했다.
- 같은 레코드의 비표시 공백 리터럴 2개도 번역 수에 포함하지 않고 원형을 유지한다.
  표시 후보와 비표시 리터럴을 분리해 검증하므로 다음 배치 경계가 흔들리지 않는다.
- SC·JP·EN·TC의 같은 block_id·record_id 레코드 전체를 대조했다. 언어마다
  리터럴 분할 수가 달라 같은 literal_id를 같은 문장이라고 가정하지 않았다.
- 저장·시나리오 편집·환경 설정 안내와 동적 이름·정책·수치 결합 문장을 번역했다.
  전 항목은 실제 화면 검수가 필요한 초벌이다.

## 산출물

- build_translation_batch5.py: 고정 설치본을 읽어 공개 산출물을 결정적으로
  생성하고 오프라인 msggame.bin을 메모리에서 재패킹·재파싱한다.
- public/msggame_ko_system_messages_b02r0666_b04r0075.v0.5.json: 공용 리터럴
  빌더와 호환되는 source-free 번역 오버레이.
- evidence/translation_alignment_evidence.v0.5.json: 언어별 같은 레코드의
  구조와 원문 해시만 기록한 정렬 근거.
- review/translation_review_index.v0.5.json: 좌표별 초벌·사람 검수·런타임 검수
  상태와 동적 결합 주의점을 기록한 검수 목록.
- translation_validation.v0.5.json: 재현성, 바이너리 구조, 문자열 불변 조건,
  설치본 불변과 배포 안전성 검사 결과.

## 재생성

~~~powershell
python -B workstreams/msggame/build_translation_batch5.py `
  --out-root workstreams/msggame
python -B -m unittest workstreams.msggame.tests.test_translation_batch5 -v
~~~

생성기는 설치된 SC·JP·EN·TC 파일의 크기와 packed/raw SHA-256을 먼저 확인한다.
고정 설치본과 다르면 쓰기 전에 중단하며 설치 경로에는 어떤 파일도 쓰지 않는다.

## 검증한 항목

- 오버레이 A/B/final 생성 결과의 바이트 일치
- 오프라인 재패킹 바이너리 A/B/final SHA-256 일치
- 18개 최상위 블록, 21,581개 레코드와 모든 리터럴 좌표 유지
- 표시 후보 153개 중 비언어 형식 3개를 보존하고 언어 리터럴 150개만 교체
- 같은 레코드의 비표시 공백 2개 원형 유지
- 이전 v0.1~v0.4 오버레이 600개 좌표와 중복 없음
- 가변 길이 한국어 적용 뒤 블록·레코드 오프셋 재계산과 4바이트 정렬 확인
- 리터럴 밖 불투명 바이트코드의 바이트 일치
- 줄바꿈, 앞뒤 공백, printf 토큰, PUA 코드포인트와 괄호·인용부호 순서 유지
- 공개 JSON의 한자 통합영역·가나 원문 노출 0
- 설치된 SC·JP·EN·TC 파일의 작업 전후 SHA-256 일치

## 남은 검수

동적 이름·정책·수치가 리터럴 사이에 삽입되는 문장은 실제 화면에서 조사와 결합을
확인해야 한다. 환경 설정의 긴 도움말도 줄바꿈과 글자 넘침을 확인해야 한다. 공개
오버레이의 150개 항목은 모두 사람 검수 필요, 런타임 검수 0개로 표시되어 있다.
