# MSGGAME B11~B13 직접 PC 감사 v1

## 결론

- 검사 범위는 Base/PK의 B11~B13이며, 직접 PC Korean 파일과 정확한 pristine PC Japanese 원본 네 개만 대조했다.
- 고신뢰 수정 후보는 **B13의 8개 리터럴 슬롯**이다. 모두 같은 메뉴·기능의 한국어 표기가 서로 달라 플레이어가 실제 UI를 찾기 어렵게 하는 문제다.
- B11에는 비어 있지 않은 대사가 없고, B12에서는 고신뢰 의미·용어 오류를 확정하지 않았다.
- 이 감사는 후보 파일, Steam 적용, Git, 네트워크, 커밋, 푸시, 릴리스를 만들거나 수행하지 않았다.
- B11~B13의 구조·용어·의미 이상을 정적으로 점검한 결과일 뿐, 전체 번역의 문학적 완결이나 실제 게임 화면 합격을 선언하지 않는다.

좌표는 `블록:레코드:리터럴` 형식이다.

## Hold — 정적 데이터만으로 수정하지 않을 항목

1. **B12/B13의 비텍스트 control 차이**
   - B12는 Base/PK 모두 레코드 `18, 21, 24, 26, 28, 45~62`의 23개, B13은 Base/PK 모두 29개에서 JP와 opaque control skeleton이 다르다.
   - B13 토폴로지 차이 16개와 겹치는 부분이 있다. 리터럴 수·런타임 문법 결합·실제 출력에 영향을 줄 수 있으므로, raw 정적 비교만으로 삭제·삽입하지 않는다.

2. **B13 줄바꿈 재배치**
   - B13의 KO/JP 수동 LF 수가 다른 슬롯은 Base 15개, PK 16개이며, 위 토폴로지 차이와 같은 대사군에 집중되어 있다. PK 전용 `13:36:0`은 3줄 JP를 2줄 KO로 재배치한 사례다.
   - Base/PK `13:213:0`의 `가능해\n집니다`처럼 한국어 어절 안에서 끊기는 줄도 확인했다. 문장만 합치면 화면 폭을 넘길 수 있으므로, 해당 UI 폭과 실제 렌더링 확인 뒤에만 재배치한다.

3. **추가 용어 정규화**
   - 같은 `勲功`에 `훈공`과 `공훈`, 같은 `制度改新`에 `제도 개신`과 `제도 개혁`, 같은 `退き口`에 `퇴로`와 `퇴각 지점`이 공존한다.
   - 의미를 뒤집는 오류로는 확정하지 않았다. 실제 HUD·메뉴의 정식 표기를 먼저 확인해 한 용어군씩 정규화해야 한다.

4. **이미 수정된/문체 판단 항목**
   - `小姓`은 B13에서 `시동`으로 표기되며 `소성` 잔재는 찾지 못했다.
   - PK `13:143:0`의 `역부족`·`힘` 반복은 문체상 거칠 수 있으나, pristine JP의 `力不足` 및 `力を認めさせる`과 의미 방향이 맞아 고신뢰 오역으로 올리지 않았다.
