# 장수명 v0.1 RC 검수 기록

검수일은 2026-07-14(KST)이다. 대상은 이 작업 공간의 비 Steam 설치본이며, 배포 방식은
EXE/DLL·레지스트리·메모리 패치를 사용하지 않는 네 리소스 파일 패치다.

## 네 리소스 recipe E2E

`tmp/officer_names_e2e`의 격리 게임 루트에서 다음을 확인했다.

- 네 순정 파일에서 최종 네 target을 한 거래로 적용하고, 다시 네 순정 해시로 원복했다.
- 기존 3명 장수명 probe + Font-v4 네 파일 벡터를 정확히 식별해 순정으로 정규화한 뒤 최종
  target을 적용하고 원복했다.
- 상위 migration 저널의 `prepared` 상태에서 predecessor snapshot 네 개를 복구한 뒤 순정으로
  원복했다.
- `cleanup_pending` 상태에서 snapshot 두 개만 남은 부분 정리도 재개해 순정으로 원복했다.
- 각 경우 종료 후 거래·migration 임시 파일은 0개였으며, 임의 혼합 해시 벡터는 허용하지 않았다.

`inputs/four_resource_recipe_e2e.json`은 위 시험에 사용한 네 stock/recipe/target SHA-256을
`final_pins.json`과 동일하게 고정한다.

## 실제 게임 runtime QA

최종 네 target을 설치한 뒤 `F:\Games\NOBU16`을 작업 폴더로 `NOBU16PK.exe`를 실행했다.
직접 실행 도우미가 다른 작업 폴더를 사용한 선행 음성 시험에서는 `ERROR:-9001`이 재현됐지만,
게임 루트에서 실행한 승인 세션에서는 이 오류가 발생하지 않았다. 설치 런처의 설치 확인 실패는
비 Steam 설치본 특성으로 패치 판정에서 제외했다.

승인 세션에서 다음을 확인했다.

- 게임 로고와 장수 편집 화면까지 정상 진입했다.
- 장수 목록·탭·능력치 항목의 한글이 공백, tofu, 겹침 없이 표시됐다.
- `오다 노부나가`가 성 `오다`, 이름 `노부나가`, 우측 전체 이름 `오다 노부나가`로 표시됐다.
- 목록에서 `아츠지 사다유키`, `안요지 우지타네` 등 한글 장수명이 정상 렌더링됐다.
- 변경 수는 `0/2216`으로 유지돼 검수 과정에서 사용자 데이터를 수정하지 않았다.
- 종료 확인 창에서 정상 종료했으며 종료 뒤 네 설치 파일 SHA-256이 최종 target과 일치했다.

상단 중국어 제목, 아직 번역하지 않은 일반 UI/특성 문자열과 장수명 범위 밖의 추가 인물은 이번
장수명 RC의 합격 범위가 아니다. 긴 이름과 전체 2,207개 문자열은 Font-v5 정적 검증에서 필요한
한글 125자를 포함해 누락 0자로 확인했으며, 이후 사용자 실기 검수 대상으로 남긴다.
