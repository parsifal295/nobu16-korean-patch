# Wave 34 Base 이벤트 실게임 QA 상태

아래 Base `ev_strdata` 변경은 선언된 수동 줄 수와 글꼴 advance를 기록했지만,
Base 이벤트에 PK `MSGEV`의 912px 제한을 적용할 근거는 없다. PC UI 리소스에서
`Text_message`의 448×100 logical px 슬롯은 확인됐으나 renderer scale·자동
줄바꿈·최대 줄 수는 정적으로 입증되지 않았다. 따라서 후보 생성·바이트 검증과
실게임 QA를 구분한다.

읽기 전용으로 Base/PK `FLOW` N160 payload 1,341개, `SCENARIO`, `PARAM`,
`SUBMIT`, PC 비메시지 리소스, 로컬 및 Steam Cloud `.n16` 세이브 68개를 각 ID의
직접 little-/big-endian 참조로 대조했다. 입증 가능한 직접 참조·이벤트 직전
세이브·시나리오 경로는 0건이다. 이는 이벤트가 발생하지 않는다는 뜻이 아니라,
현재 자료만으로 재현 경로를 주장할 수 없다는 뜻이다.

실게임 QA를 하려면 해당 이벤트 직전 세이브 또는 재현 가능한 시나리오 조건이
필요하다. 후보를 적용한 뒤에는 전체 메시지 박스와 세 줄이 보이는 스크린샷으로
확인한다. 해상도를 바꿨다면 `NOBU16PK.exe`를 완전히 종료·재실행한 뒤, 선택한
해상도와 재시작 여부를 함께 기록한다.
