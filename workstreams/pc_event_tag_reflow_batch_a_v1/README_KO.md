# PC 이벤트 색상 태그 개행 재검토 A

## 범위와 근거

- 대상: Steam PC PK 이벤트 W45 입력의 정확히 10개 ID
  (4057, 4257, 4323, 4436, 4726, 4737, 4792, 4880, 4895, 5182).
- 한국어 입력: F:\SteamLibrary\steamapps\common\NOBU16\MSG_PK\JP\msgev.bin
  - packed SHA-256: 01287E2ECC5328C85348657EFF06553353CB8664B0FB7E1669DB9FC591D53EBE
  - raw SHA-256: F3716AB98D60931CEC0FE61976D8DAD386C05B30B7167BD1BDB2CDF02EC55ACC
- 일본어 근거: PC 백업의 MSG_PK/JP/msgev.bin만 사용했다.
  - packed SHA-256: 03426B59D32EB628021DE43BC02FF82B56B04D97C25CE37F735630EA7C4E2002
  - raw SHA-256: 4A916CA6837C4F8FC2D8B6254ECBEF26339558D2DDFEBF5A1637F8426F5918DE
  - 기존 Steam↔legacy PC JP 매핑
    (msgev_ko_steam_jp_native.v1.json,
    SHA-256 47742330C4375A6BB6AC19ED0F7E8040CF57E22EF39BEDEE7FF4959520B1575C)으로
    10개 모두 Steam ID = JP ID임을 확인했다.
- Switch 파일·번역은 읽거나 검색하지 않았다. Steam, Git, 네트워크, 릴리스에는 접근하거나 쓰지 않았다.
- 폭은 실제 이벤트 글꼴 RES_JP/res_lang.bin
  (3798CB758E6EA48A257F1FBBBBE56E800F668E6FA2DE0CFD4B277C785A322EE7)로 측정했다.

표기: 아래의 "\x1bC?"와 "\x1bCZ"는 실제 색상 이스케이프 바이트를 사람이 읽을 수 있도록 적은 것이다.
줄은 실제 수동 개행 위치를 그대로 보여 준다.

공통 검증 결과: 모든 권고안은 색상 태그 순서·런타임 토큰·제어 토큰을 그대로 유지하고,
색상 span 안의 LF를 1 → 0으로 줄였으며, 1~3줄 및 줄당 ≤ 912 px를 만족한다.
이 문서는 **검토 보고서만**이며 후보 파일이나 게임 파일을 만들지 않는다.

## 인수 전 확인 항목

- 10개 권고안은 모두 현재 W45 입력의 단일 색상 span 내부 LF를 제거한다.
- 태그/런타임/제어 토큰 시그니처는 각 현재 레코드와 동일하다.
- 권고안은 아직 후보·합성본·Steam 파일에 반영되지 않았다. 이후 후보를 만들 때도 이 보고서의 10개 ID만 바꾸고, W45 입력 고정과 실제 게임 재시작 후 화면 QA를 별도로 수행해야 한다.
