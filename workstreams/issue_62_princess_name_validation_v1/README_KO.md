# 이슈 62: 가공 히메 작명 검증 정적 EXE 패치

## 원인

Steam JP 1.1.7 실행본에서 이슈 43의 무장 에디트 검증 함수는 `RVA
0x00BB0110`이지만, 가공 히메 작명 화면은 별도 함수 `RVA 0x00EB4660`을
사용한다. 두 함수는 공통 문자 검증기 `0x00B9B610` 및 `0x00B9B6B0`을
직접 호출한다. 기존 이슈 43 패치는 무장 에디트 함수 안의 다섯 분기만
수정하므로 가공 히메 경로에는 적용되지 않았다.

## 수정

가공 히메 함수 안의 네 직접 호출을 동일 길이의 `mov eax, 1`로 교체한다.
공통 문자 검증기는 수정하지 않는다.

| 대상 | RVA | 언팩 EXE 오프셋 | 원본 | 패치 |
|---|---:|---:|---|---|
| 입력 이름 | `0x00EB47BF` | `0x00EB3BBF` | `E8 4C 6E CE FF` | `B8 01 00 00 00` |
| 입력 독음 | `0x00EB47E0` | `0x00EB3BE0` | `E8 CB 6E CE FF` | `B8 01 00 00 00` |
| 상속된 성 | `0x00EB484F` | `0x00EB3C4F` | `E8 BC 6D CE FF` | `B8 01 00 00 00` |
| 상속된 성 독음 | `0x00EB4878` | `0x00EB3C78` | `E8 33 6E CE FF` | `B8 01 00 00 00` |

호출 앞의 길이 분기와 호출 뒤의 공란·합산 길이 검사는 유지된다. 따라서
이름과 독음이 비어 있으면 계속 오류가 나며, 일본어형 성+이름 합계 제한도
그대로 적용된다. 문자 종류 검사만 이 화면에서 성공 처리된다.

## 패치 계약

- 원본 Steam JP 1.1.7:
  `29BC1ED66D27B9AEF5EB6CE3D126BA2BDBF86099E12B09615FE9F988F41E2246`
- Steamless 언팩 원본:
  `BC885875A5E4288E5A1A424D99974F6F215777C03569C7EA707FDE63BDBC2B39`
- 기존 이슈 43 패치본:
  `2E098ECB5E4335DC264F865306B990B724EA7C242B1B9F87FFC5EE2E7191797C`
- 이슈 43+62 통합 패치본:
  `7CA2F1D59E02650C67F343F0776F6D05517C0486B65168E63A9AE4CBCAAFDBB2`
- 출력 크기: `31,747,848`
- PE 체크섬: `0x01E51698`
- 총 패치 지점: 9개(이슈 43 다섯 곳 + 이슈 62 네 곳)

이전 패치본과 통합 패치본의 바이트 차이는 이슈 62 코드 20바이트와 PE
체크섬 필드 3바이트뿐이다.

## 배포

과거 릴리스 payload를 변경하지 않도록 `release_payload/v0.11.5`를 새로
만들었다. 설치기는 다음 두 입력을 모두 지원한다.

1. 원본 wrapped EXE: Steamless로 언팩한 뒤 아홉 지점을 적용한다.
2. 기존 이슈 43 패치 EXE: 기존 다섯 지점의 postimage를 확인하고 이슈 62
   네 지점만 추가한다.

두 경로 모두 최종 해시, 사전 바이트, 함수 prefix, PE 체크섬을 검증한 뒤
원자적으로 `NOBU16PK.exe`를 교체한다. 원본 복구와 최신 패치 멱등 실행도
지원한다.

## 자동 검증 결과

- 원본 EXE 신규 설치: 통과
- 기존 이슈 43 패치본 인플레이스 업그레이드: 통과
- 최신 패치 재실행: 변경 없이 통과
- 원본 복구: 두 설치 경로 모두 통과
- 독립 디스어셈블: 네 지점 모두 `mov eax, 1` 확인
- v0.11.5 관련 회귀 테스트: 51개 통과

## 실게임 QA

Steam 라이브 경로에 v0.11.5와 동일한 설치기 계약을 적용했다. 적용 전 게임 프로세스가
종료된 것을 확인했으며, 패치 후 새 프로세스로 완전히 재실행했다.

- 선택 해상도: `3840x2160` (`window mode : 1`)
- 전체 프로세스 재시작: 완료
- 라이브 EXE SHA-256:
  `7CA2F1D59E02650C67F343F0776F6D05517C0486B65168E63A9AE4CBCAAFDBB2`
- 사용자 실게임 확인: 정상 동작

현재 구현은 한글 허용목록을 추가한 것이 아니라 이 화면의 문자 종류 검사 네
곳을 성공 처리한다. 따라서 임의의 한글 이름과 독음도 입력할 수 있다. 공란
검사와 기존 길이 제한은 그대로 유지된다. 다음 항목 중 금지 문자 오류 해소와
결정 가능 여부는 확인되었고, 저장 후 유지 여부는 별도 장기 검증 항목이다.

1. 가공 히메 이름과 독음에 한글을 입력하면 금지 문자 오류가 사라진다. (확인)
2. 이름 또는 독음이 공란이면 기존 오류가 유지된다. (코드 경로 및 자동 검증)
3. 허용 길이 안의 이름에서 결정 버튼이 활성화되고 이벤트가 종료된다. (확인)
4. 저장 후 가공 히메의 성명과 독음이 유지된다.
