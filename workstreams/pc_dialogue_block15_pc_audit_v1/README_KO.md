# Steam PC 인물 대사 블록 15 재감사

PC 일본어 원문을 기준으로 현 W45 Steam 한국어 `msggame.bin`의 블록 15를
다시 대조한 읽기 전용 감사다. Switch 자료는 열거나 판정에 사용하지 않았다.

## 범위와 무결성

| 구분 | 범위 | 원문 packed SHA-256 | 결과 |
| --- | --- | --- | --- |
| Base | `B15/R0000–R2422`, 2,423 records | `EDEC6E21FE663A815422A16C219C3429262606ECADA8E814F2E9864250A463C4` | JP 4,489 / KO 4,485 literal slots |
| PK | `B15/R0000–R2600`, 2,601 records | `31D52FB797EA31CBD75646A2E1607829635AC51C288606FB2ADFBDCA940F4210` | JP 5,039 / KO 5,035 literal slots |

- Base 원문은 `F:\Games\NOBU16\MSG\JP\msggame.bin`, PK 원문은 Steam JP
  backup의 `MSG_PK\JP\msggame.bin`만 사용했다.
- 네 입력 모두 raw parse/rebuild byte-exact 및 wrapper semantic round-trip을
  통과했다.
- PC EN/SC/TC는 문맥 보조로만 사용했다.
- KO block 15의 수동 LF 포함 record는 Base 2,014개, PK 2,140개이며,
  runtime/control이 섞인 record는 Base 1,402개, PK 1,564개다.

이 범위는 구조와 원문 대조를 전량 수행하고, 확실한 의미·용어 오류를
추출한 감사다. 모든 대사를 사람이 문학적으로 완독했다는 선언은 아니다.

## 정적 후보

정적 SAFE 후보는 **0건**이다. 확인된 오류는 모두 runtime slot, `02xx`/`0143`,
수동 LF, 또는 실제 UI 폭이 얽혀 있어 별도 실게임 QA 후보로만 유지한다.

| 좌표 | 현 KO → 제안 KO | PC JP 근거 | 보류 사유 |
| --- | --- | --- | --- |
| Base `15:1121:2` | `이 절연` → `의 관계가 단절되었습니다.` | `が手切` | 2 runtime slots, UI bullet, 폭 증가 |
| Base `15:1875:1`, `1890:1` | `마술 훈련` → `승마 훈련` | `馬術の訓練` | 수동 LF + runtime |
| Base `15:1899:2`; PK `15:1929:2` | `보고할 수 없` → `보고할 수 있` | `報告でき` | 수동 LF + runtime; 의미 반전 |
| PK `15:2257:1`, `2258:1` | `다음 성을 함락했습니다:` → `를 함락시켜라` | `を落とせ` | 성명 slot + 수동 LF |
| PK `15:2279:0` | `침공할 수 있어` → `침공할 수 없어` | `侵攻することができ`의 부정 문맥 | 수동 LF; EN/SC/TC도 cannot/unable |
| Base `15:1096:3`; PK `15:1104:3` | `심증이 악화` → `인상이 악화` | `心証` | runtime + 수동 LF |
| Base `15:1206:2`; PK `15:1214:2` | `심증이 이전보다 좋아졌다` → `인상이 이전보다 좋아졌다` | `心証` | runtime + 수동 LF |
| Base `15:1657:0`, `1659:0`, `1661:0`; PK `15:1687:0`, `1689:0`, `1691:0` | `심증은 충분하니` → `호감은 충분하니` | `心証` | runtime/control + 수동 LF |

일본어 `心証`은 이 문맥에서 대인 인상·호감을 뜻한다. 한국어 법률어
`심증`으로 옮기면 의미가 달라지므로, 앞 두 계열은 `인상`, 동맹 권유 계열은
`호감`으로 분리했다.

## 기존 보류의 재확인

아래 항목도 원문 근거는 있으나 정적 후보로 승격하지 않았다.

| 좌표 | 검토한 대안 | JP 근거 | 보류 사유 |
| --- | --- | --- | --- |
| PK `15:1520:2`, `1521:0`, `1522:0`; Base `15:1505:2`, `1506:0`, `1507:0` | `조장` → `조두` | `組頭` | `0143`/`02xx` 또는 command topology |
| PK `15:2060:1`, `2144:1`; Base `15:2030:1`, `2114:1` | `용서` → `허락` | `をお許しください` | runtime 삽입값과 화면 폭 QA 필요 |
| PK `15:2161:1`; Base `15:2131:1` | `용서` → `허락` | `をお許しあれ` | `026432`, `023c` runtime token |
| PK `15:2285:1`; Base `15:2254:1` | `헌언의 용서` → `헌언의 허락` | `献言の許し` | `0143` command topology |

수동 LF가 단어 안에 있는 record도 자동 병합하지 않는다. 문장 의미와 실제 UI
폭을 확인한 뒤에만 별도 후보로 다룬다.

이 작업물은 Steam 파일, backup, Git 원격, 푸시, 릴리즈를 변경하지 않는다.
