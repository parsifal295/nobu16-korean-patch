# Wave 37 — PC 동적 세력·항목 대사 보정

현재 Steam PC의 런타임 슬롯 대사 14좌표를 private 후보로만 보정한다.

- Base `6:625, 631, 634, 636, 643, 645, 647`: 동적으로 삽입되는 세력명 뒤에 `이(가)`·`은(는)`이 그대로 출력되는 문제를 ` 측이` 형식으로 바꾼다. 원문의 ‘당가를 노림/침공’ 의미도 자연스럽게 회복한다.
- PK `6:2700`: 동적 세력명과 조사의 결합 및 원문의 양보절을 고친다.
- Base `6:1407, 1408, 1411` 및 PK `6:1411, 1412, 1415`: 같은 군단 조정 UI의 동적 항목 슬롯 `023C` 뒤에 고정 조사가 붙는 문제를 없앤다. 특히 PK `6:1411`의 “인계 불가”는 원문의 ‘넘겨줄 수 있는 항목이 없음’과 정반대이므로 함께 바로잡는다.
- 근거는 PC Base/PK JP 및 가능한 PC EN/SC/TC만 사용한다. Base EN은 설치본에 없으며 Switch 자료는 읽지 않는다.
- 모든 레코드의 opaque 런타임 opcode, marker, 종료 코드, 수동 개행 수를 byte-identical로 보존한다. 활성 글꼴의 정적 literal 폭은 3줄·912px 이내다.

런타임 세력명·항목값의 실제 폭은 아카이브만으로 확정할 수 없다. 따라서 이 후보는 **Steam 적용 전 실게임 QA가 필수**이며, 산출물은 `tmp/pc_dialogue_quality_wave37_runtime_slots_v1/candidate/` 아래에만 만든다. Steam 적용, Git stage/commit/push, GitHub, 릴리즈 기능은 없다.

특히 PK `6:2700`의 첫 줄은 정적 literal만으로 744px이며 동적 세력명에 남는 폭이 168px뿐이다. 실제 세력명 길이에 따라 자동 줄바꿈이 달라질 수 있으므로, 이 좌표는 화면 QA를 통과하기 전에는 적용하지 않는다. `023C` 항목 슬롯도 실제 값 종류·표시 폭을 정적으로 확정하지 못했으므로, 군단 조정 화면에서 화면 QA를 통과하기 전에는 적용하지 않는다.

Base `9:1996` / PK `9:2070`은 외부 이름이 실제로 결합되는지 정적 근거와 재현 경로가 없어 변경하지 않는다.
