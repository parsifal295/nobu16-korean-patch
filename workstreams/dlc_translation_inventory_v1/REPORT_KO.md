## 결론

이번에 실제로 새로 들어온 번역 대상은 **일본어 컨테이너 77개, 화면 노출 레코드 670개**다. 동일한 일본어 원문을 정확히 중복 제거하면 **662개 원문**이며, 일본어 내부 코드 단위는 배치 기준 27,971개, 중복 제거 기준 27,947개다.

현재 DLC 전체를 처음부터 완성한다고 보면 `DLC/JP`와 `DLC_PK/JP`의 **105개 파일, 881개 적용 위치**를 다뤄야 한다. 원문 중복을 제거한 실제 번역 작성량은 **786개**다. 이 중 이번 신규분이 662개이고, 기존 데이터에만 있는 원문이 124개다.

따라서 작업 기준은 다음과 같이 확정한다.

1. 번역 작성 기준: 전체 786개 원문, 이 가운데 신규 우선 662개
2. 패치 적용 기준: 전체 105개 파일의 881개 레코드, 이 가운데 신규 우선 77개 파일의 670개 레코드
3. 이벤트 레코드는 기존 PK 이벤트 대사 레이아웃 규칙으로 별도 줄바꿈 검사를 수행

## 제외 기준

XL13 테이블에는 예약 빈 칸과 화면에 표시하지 않는 메타데이터가 포함돼 있다. 단순 슬롯 수 7,104개를 번역량으로 잡지 않고 아래 순서로 제외했다.

- 빈 레코드 제외: 7,104개 슬롯 → 비어 있지 않은 레코드 791개
- `evm` 257번: 언어별 정렬키이므로 제외
- `evm` 258–267번: 이벤트 발생조건용 내부 템플릿이므로 제외
- `tom` 1번: 언어별 정렬키이므로 제외
- 내부 메타데이터 제외 121개: 791개 → 화면 번역 대상 670개

`evm` 256번 이벤트 제목은 화면 표시 항목이므로 포함했다. 제어코드·색상 태그·동적 토큰이 들어간 이벤트 본문도 레코드 수에는 포함하되, 실제 번역 시 원형을 보존해야 한다.
