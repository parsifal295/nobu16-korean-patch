# PK 이벤트 문장 직접 번역 3차

`MSG_PK/SC/msgev.bin`의 정확 대상 키 가운데 기존 35개 오버레이가
차지하지 않은 다음 안전 의미 문장 350개를 한국어로 번역한 독립 작업선이다.
루트 진행률, 루트 README, 글꼴, 설치본은 수정하지 않는다.

## 범위

- 정확 대상 전체: 12,906개
- 선행 오버레이 완료: 11,580개
- 작업 전 잔여: 1,326개
- 이번 번역: 350개
- 작업 후 잔여: 976개
- 선택 범위: 9,938번부터 10,291번까지의 미점유 안전 의미 행
- 누적 안전 제외: 136개
- 이번에 새로 확인한 구두점 전용 제외: 10,105번, 10,258번

내부 키, 화자 키, 동적 치환 토큰, 구두점 전용 행은 번역 진척으로
계상하지 않고 원본 값을 보존한다. 이번 350개에는 사용자 정의 대괄호
치환 토큰이 없으며, 모든 행에서 ESC 순서와 개수, 줄바꿈, 사용자 정의
토큰 계약을 원본 간체중문 기준으로 검증한다.

## Switch 재사용 판정

Switch v1.3 자료를 먼저 조회했다. 선택된 350개에는 일본어 원문이
정확히 일치하면서 의미 있는 한국어를 제공하는 행도, 같은 인덱스에서
의미가 정렬되는 행도 없었다. 따라서 이번 배치는 Switch 문구를 억지로
이식하지 않고 PK판의 일본어·간체중문·번체중문·영어 정렬 문맥으로 직접
번역했다. 이 판정은 빌드 때마다 다시 계산한다.

## 산출물

- `public/msgev_ko_pk_native_batch03_350.v1.json`: 원문 없는 350개 오버레이
- `evidence/msgev_pk_native_batch03_alignment.v1.json`: 네 언어 원문 해시와 선택·제외 근거
- `review/msgev_pk_native_batch03_review.v1.json`: 번역 및 런타임 검토 상태
- `validation.v1.json`: 재구축·결정성·안전 검증 결과
- `translations.py`: 한국어 번역표

## 재현과 검증

```powershell
python -B workstreams/msgev_pk_native_batch03/build_msgev_pk_native_batch03.py
python -B -m unittest discover -s workstreams/msgev_pk_native_batch03/tests -p "test_*.py" -v
```

빌더는 고정된 순정 SC 파일에서만 후보를 만들고, 격리된 A/B 빌드와 최종
빌드의 바이트 동일성을 확인한다. 자기 오버레이 등록 전후 결과가 같고,
후속 오버레이가 등록되어도 고정 선행 집합과 선택 결과가 바뀌지 않는지
검증한다. 17,910개 전체 행을 재파싱하여 선택하지 않은 행이 순정 값과
동일한지도 검사한다.

상용 원문 문자열이나 완전한 게임 리소스는 배포 산출물에 포함하지 않는다.
