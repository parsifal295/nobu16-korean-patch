# W75 도요토미 이벤트 품질·개행 검수

이 작업 스트림은 PK 이벤트 `8484–8520` 37행을 검수한다. 입력은 반드시
W74의 **온디스크** 후보 파일
`tmp/pc_event_honnouji_quality_wave74_v1/candidate-final/MSG_PK/JP/msgev.bin`이며,
W74 재구성 결과·온디스크 바이트·압축/원본 프로필이 모두 고정값과 일치하지
않으면 빌드를 중단한다.

수정 행은 정확히 다음 15행이다.

`8491, 8496, 8498, 8499, 8503, 8506, 8508, 8510, 8512, 8513, 8514, 8515, 8516, 8519, 8520`

나머지 22행은 검수 후 그대로 유지하며, 특히 `8495`는 수정 대상이 아니다.
직접 PC 일본어 원문과 제어 코드 구조를 기준으로 검수했으며, 일본어 원문의
수동 개행은 이식하지 않는다. 한국어 문장을 축약·삭제하지 않고, 수동 개행은
한국어 의미 단위에서만 배치한다. 태그·런타임 토큰·종료 구조는 보존하고 태그
안에 개행을 넣지 않는다.

레이아웃은 최대 4줄, 원본 G1N raw 폭 960px 이하를 강제한다. 보고서에는 각
표시 줄의 문자열·raw 폭·`ceil(raw * 30 / 48)` 실효 폭·전각/반각 수·초과 여부를
기록한다. 동적 토큰은 전역 표시값으로 단정하지 않고, 이 장면에서 필요한 더 긴
이름 별칭을 이용한 보수적 용량 상한으로 측정한다.
