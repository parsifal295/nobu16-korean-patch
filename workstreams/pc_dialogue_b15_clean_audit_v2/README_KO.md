# PC MSGGAME B15 클린 재감사 v2

## 결론

PC Base/PK 한글 바이너리와 각각의 PC 일본어 원본만 같은 좌표로 직접 대조했다. 정적 근거가 충분한 것은 5개 유형, 13개 레코드, 15개 리터럴 교체 방향이다. 이 보고서는 후보 파일이나 바이너리를 만들지 않았고, 어떤 수정도 적용하지 않았다.

이 결과는 B15의 정적 의미 대조 결과다. 전체 문학 품질 완료, 런타임 토큰 결합, 실제 화면 줄바꿈 또는 폭 안전성을 보증하지 않는다. 그런 근거가 필요한 항목은 모두 Hold로 남겼다.

## 고신뢰 정적 오류와 수정 방향

아래는 일본어 원문과 뜻이 직접 충돌하거나, 한글 문장이 명백히 깨지는 경우만 남긴 것이다. 제안 문구는 방향을 보이기 위한 것이며 이 보고서에서 파일에 쓰지 않았다.

### 1. 馬術을 마술로 번역한 Base 전용 오역

- 좌표: Base `15:1875:1`, `15:1890:1`
- 현 KO: `↵마술 훈련을 실시해↵병사를 강화하고`
- JP: `↵馬術の訓練を行い↵兵を強化し`
- 판정: 馬術은 승마·마술이 아니라 마술이 아니다. 같은 PC JP 원문에 대응하는 PK `15:1905:1`, `15:1920:1`은 이미 `승마 훈련`으로 되어 있다.
- 방향: 두 Base 리터럴의 `마술`만 `승마`로 교체하고 현재 개행은 유지한다. 대상 2 슬롯.

### 2. 城将을 성장으로 번역한 오역

- 좌표: Base `15:1460:0`, PK `15:1475:0`
- 현 KO: `공성이란 성장의 마음을 치는 것↵`
- JP: `城攻めとは城将の心を攻めるもの↵`
- 판정: 이 문맥의 城将은 성을 지키는 장수·성주이며, 成長이 아니다.
- 방향: `성장`을 `성주`로 바로잡아 `공성이란 성주의 마음을 치는 것`이 되게 한다. 대상 2 슬롯.

### 3. お許し를 용서로 번역한 의미 반전

- 좌표 A: Base `15:2030:1`, `15:2114:1`; PK `15:2060:1`, `15:2144:1`
- 현 KO / JP: `을 용서해 주십시오` / `をお許しください`
- 방향 A: 행위를 용서해 달라는 뜻이 아니라 허락해 달라는 요청이므로 `을 허락해 주십시오` 방향. 대상 4 슬롯.

- 좌표 B: Base `15:2131:1`; PK `15:2161:1`
- 현 KO / JP: `을 용서하시오` / `をお許しあれ`
- 방향 B: `을 허락하시오` 방향. 대상 2 슬롯.

### 4. 위협 + 자 리터럴 이음부의 명백한 문법 파손

- 좌표: Base `15:1434:0-1`
- 현 KO: `의 대군은 위협` + `자↵저 성의 발을 묶기 위해↵불을 놓아 군량을 빼앗음이 어떻겠소?`
- JP: `の大軍は脅威` + `な↵かの城の身動きを封じるため↵焼討にて軍糧を奪ってはいかが？`
- 판정: 현재 리터럴 이음부는 `위협자`가 되어 문법과 뜻이 무너진다. 동일 PC JP 대사에 대한 PK `15:1449:0-1`은 앞의 `자` 없이 자연스러운 후문으로 정리되어 있다.
- 방향: 최소한 두 번째 리터럴 선두의 `자`를 제거하고, `위협입니다만`으로 문장을 닫은 뒤 성의 움직임을 막고 군량을 빼앗자는 제안으로 다시 연결한다. 실제 후보 작성 전에는 토큰 결합과 화면 확인이 필요하다. 대상 1 슬롯.

### 5. 献言の許し를 헌언의 용서로 옮긴 오류

- 좌표: Base `15:2254:0-1`; PK `15:2285:0-1`
- 현 KO: `헌언의` + `용서,`
- JP: `献言の` + `許し、`
- 판정: 문맥은 헌언·진언을 해도 된다는 허락이지, 용서가 아니다. 현재 결합문 `헌언의 용서`는 한국어로도 부자연스럽다.
- 방향: 두 리터럴을 보존한 채 `헌언을` + 허락을 뜻하는 연결어로 바꾼다. 예: `헌언을` + `허락하신다면,`. 대상 4 슬롯.

## 정적 검증과 Hold

### 리터럴 구조

KO와 JP의 B15 레코드 수는 각각 같지만, 아래의 초기 대사에서 한글 리터럴이 합쳐져 있다. 뜻·폭·토큰 결합을 정적으로 확정할 수 없으므로 리터럴 개수만 맞추려는 수정은 하지 않는다.

| 파일 | KO / JP 리터럴 수가 다른 좌표 |
| --- | --- |
| Base | `15:250` 1 / 2, `15:255` 2 / 3, `15:257` 1 / 3 |
| PK | `15:253` 1 / 2, `15:258` 2 / 3, `15:260` 1 / 3 |

### 수동 LF 수 차이

각 리터럴의 LF 개수만 비교했다. 아래 10개는 줄바꿈 정책이나 런타임 표시 폭을 실제 게임에서 확인하지 않고 수정하면 안 된다.

| 파일 | 좌표와 KO / JP LF 수 |
| --- | --- |
| Base | `15:250:0` 1 / 0, `15:255:0` 1 / 0, `15:257:0` 2 / 0, `15:271:0` 1 / 2, `15:278:0` 1 / 2 |
| PK | `15:253:0` 1 / 0, `15:258:0` 1 / 0, `15:260:0` 2 / 0, `15:274:0` 1 / 2, `15:281:0` 1 / 2 |

### 제어 스켈레톤

## 문자 잔재 및 알려진 이상 문자열 확인

검사는 리터럴 단위로 했다. B15 KO에는 CJK 한자, U+FFFD, NUL, ESC가 없었다.

- Base KO에서 일본어 블록에 속하는 문자는 U+30FB `・` 5개뿐이었다: `15:710:2`, `15:770:2`, `15:771:2`, `15:1122:1`, `15:1122:2`. 모두 상태 목록의 글머리표이며 일본어 어휘 잔재가 아니다.
- PK KO에는 일본어 가나·한자 잔재가 없다.
- 리터럴 단위 exact 검색에서 `덴령`, `상사람`, `소성`, `했습니다군`, `하였습니다군`, `나오했습니다군`은 Base와 PK 모두 0건이다. 이 값은 B15 범위에만 해당한다.
- `마술`, `용서`, `성장`, `헌언의`는 위 고신뢰 대조를 위해 따로 확인했다. 특히 `성장`은 양쪽 5건 중 4건은 JP `成長`에 맞는 정상 용례이고, `城将`에 대응하는 각 1건만 오류다.
