## 수행한 정적 검사

1. 네 입력의 크기·SHA-256 고정값을 확인했다.
2. 각 packed 파일을 풀고, 전체 archive의 raw parser 재조립 및 LZ4 재압축/해제 왕복을 확인했다.
3. B06에서 KO/JP 레코드 수와 각 레코드의 리터럴 개수를 전수 대조했다. 리터럴 토폴로지 불일치는 0건이다.
4. KO B06 리터럴 전체에서 실제 일본어 가나·한자 잔재, U+FFFD, NUL, ESC를 검사했다. 각 항목은 Base/PK 모두 0건이다.
5. 원문과 현재 KO의 수동 LF 수, 리터럴 외 제어 skeleton, 동일 JP 원문의 Base/PK 번역 차이를 대조해 고신뢰 의미 오류와 런타임 확인 대상을 분리했다.

## Hold (3)

### H-1 — `6:751:0` 의미·문체 확인

- PK KO: `주군이 지나치게 강경하면\n가신들의 사기가 꺾이지……`
- pristine JP: `主君が強すぎては\n家臣が意気消沈する…`
- `強すぎる`의 초점(주군 개인의 지나친 강함/위세)이 KO에서는 `강경하면`(정책·태도)으로 이동했고, 두 번째 줄도 `꺾이지……`에서 멈춘다.
- 문맥상 의도한 캐릭터 말투일 가능성을 배제할 수 없으므로 고신뢰 결함으로 올리지는 않았다. 같은 대사군·화자 맥락과 실제 표시를 확인한 뒤 방향을 정한다.

### H-2 — 리터럴 외 `0x01 0x43` 문법 매크로 차이

- Base 67개 레코드, PK 59개 레코드에서 KO/JP의 리터럴 외 skeleton이 다르다.
- 하지만 두 쪽 모두 리터럴 수는 같고, 해당 `0x01 0x43 <id> 00 00` 매크로만 제거하면 126개 전부 skeleton이 일치한다. 정적 파손 증거는 없다.
- 이는 일본어 활용/접속용 런타임 토큰을 한국어 문장으로 흡수한 것으로 보인다. 실제 이름·동적 문구가 누락되지 않는지는 게임 화면에서 대표 사례를 확인해야 한다.
