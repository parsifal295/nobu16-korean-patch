# PC PK 이벤트 색상 태그 개행 재검토 — Batch C

이 문서는 Steam PC PK 이벤트 W45 입력과 보존된 PC 일본어 원문을 **지정된 11개 ID만** 대조한 검토 보고서다. 후보 파일·게임 파일·Steam·Git에는 쓰지 않았다.

## 입력과 표기

- Korean W45: `MSG_PK/JP/msgev.bin`
  - packed SHA-256: `01287E2ECC5328C85348657EFF06553353CB8664B0FB7E1669DB9FC591D53EBE`
  - raw SHA-256: `F3716AB98D60931CEC0FE61976D8DAD386C05B30B7167BD1BDB2CDF02EC55ACC`
- 보존된 PC 일본어 원문: `MSG_PK/JP/msgev.bin`
  - packed SHA-256: `03426B59D32EB628021DE43BC02FF82B56B04D97C25CE37F735630EA7C4E2002`
  - raw SHA-256: `4A916CA6837C4F8FC2D8B6254ECBEF26339558D2DDFEBF5A1637F8426F5918DE`
- 이 배치에서는 각 대상의 PC JP 대응 인덱스를 같은 ID로 명시 확인했다. 이는 전체 테이블의 일반 매핑을 주장하는 것이 아니다.
- 아래 표기에서 `<ESC>`는 실제 U+001B, `<LF>`는 수동 U+000A다. 실제 후보에서는 이를 제어문자로 복원해야 한다.
- 폭은 실제 PC 이벤트 글꼴로 측정했다. 모든 권고안은 정확히 3줄, 줄마다 912px 이하이며, 모든 `<LF>`가 색상 span 밖에 있다.
- 모든 권고안은 W45 현재문과 ESC 태그 순서·런타임 토큰·printf 토큰·기타 제어문자·앞뒤 공백 signature가 동일하다.
- 후속 PC 전용 재검증에서 각 ID의 Steam↔보존 PC JP 대응, LF를 제외한 제어·태그 signature, span 내부 LF 부재와 실제 글꼴 폭을 다시 확인했다.

## 적용 전제

- 이 문서는 후보 생성물이 아니다. 병합 시에는 W45를 다시 입력으로 읽고, 위 target UTF-16LE hash와 실제 이벤트 글꼴 폭을 재검증해야 한다.
- 3960은 이름 전용 변경과 충돌하므로, 최종 통합 후보에는 이 문서의 3960 target 하나만 넣어야 한다.
- 실제 게임 적용은 통합 후보의 범위·패킹·게임 화면 QA가 끝난 뒤에만 검토한다.
