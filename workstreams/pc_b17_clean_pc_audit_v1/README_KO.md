# PC MSGGAME Block 17 독립 대조 점검

## 범위와 근거

이 보고서는 아래의 **PC 바이너리 4개만** 직접 열어 대조한 결과다. 기존 Block 17 작업물, 후보 파일, 다른 플랫폼 텍스트, 외부 명명 자료는 참조하지 않았다.

| Base 현재 한국어 | `F:\SteamLibrary\steamapps\common\NOBU16\MSG\JP\msggame.bin` | `F9342D73DE50FDFC97C1F8365A20FD5CEABD024CE63B82AF1F112D5EDEDCFCBB` | 33 records / 66 literal slots |

파서는 `workstreams\msggame\msggame_format.py`만 사용했다. Base와 PK 모두 JP/KO의 Block 17 record 수와 literal-slot 모양이 일치했다. 총 1,192 records / 2,322 JP-KO literal 쌍을 직접 대조했다.

Base 33 records에서는 이 보고서 기준으로 즉시 적용할 고신뢰 문장 수정은 찾지 못했다. 다만 `田北` 표기는 PK 안의 상충 표기와 함께 아래 보류 목록에 기록했다.

이것은 전수 **대조 점검**이다. 모든 문장의 문학적 완성도·말투·고유명사 표준화를 완료했다는 주장은 하지 않는다. 아래에는 원문 의미가 뒤집히거나, 눈에 보이는 조사가 남거나, 명백한 문법/표기 오류가 난 항목만 적었다. 폰트 폭, 실제 UI 줄바꿈, 런타임 치환값은 이 점검의 검증 대상이 아니다.

좌표는 `PK 17:<record>:<literal>` 또는 `Base 17:<record>:<literal>`이다.

## 고신뢰 발견 — 정적 후보와 런타임 보류 분리

정적 후보는 해당 literal 텍스트만 바꾸는 전제다. 줄바꿈, 이름/색상 런타임 토큰, 제어 바이트는 바꾸지 않는다. 런타임 이름의 받침을 알 수 없는 항목은 아래 표에 있어도 정적 후보에서 제외한다.

이 재분류 기준에서 정적 후보는 PK 31개 record의 31개 literal이며, `PK 17:920:0,1`은 런타임 보류다.

## 미수행 및 상태

- 이 점검은 읽기 전용이었다. Steam 파일, 후보 바이너리, Git, 네트워크, 릴리스는 변경하지 않았다.
- 이 보고서에는 적용 스크립트나 후보 파일이 없다. 위 정적 후보도 별도 레코드 단위 빌드·구조 검증·UI/런타임 QA 전에는 적용하지 않는다.
- 특히 런타임 이름이 붙는 17:920 및 목표 UI 항목은 실제 출력과 줄 폭을 함께 확인해야 한다.
