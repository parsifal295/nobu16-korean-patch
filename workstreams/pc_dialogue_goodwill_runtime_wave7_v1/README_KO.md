# PC 친선 런타임 문구 정정 — Wave 7

## 범위

Steam PC판의 `MSG/JP/msggame.bin`, `MSG_PK/JP/msggame.bin`에서 PC 원문과 현재 용어표로 확정한 12건을 정정한다.

| 리소스 | 좌표 | 사유 |
| --- | --- | --- |
| Base | `6:1518` | 거리 때문에 친선을 계속할 수 없음 |
| Base | `6:1519` | 비용 부족으로 친선을 계속할 수 없음 |
| Base | `6:1520` | 중개자 부재로 친선을 계속할 수 없음 |
| PK | `6:1524` | 거리 때문에 친선을 계속할 수 없음 |
| PK | `6:1525` | 비용 부족으로 친선을 계속할 수 없음 |
| PK | `6:1526` | 중개자 부재로 친선을 계속할 수 없음 |
| PK | `6:3887` | 시설명 런타임 토큰 뒤의 일본어 굴절 파손 |
| PK | `8:1095`, `8:1104`, `8:1111` | 사찰·신사 명칭 자리에 남은 잘못된 라벨 |
| PK | `8:1178`, `8:1180` | 명찰·신사 이름 주변 문장/줄바꿈 파손 |

현재 한국어 레코드는 일본어 전용 `0143` 굴절 명령을 그대로 남겨, 예를 들어 `친선가`, `이(가)`, `계속될`처럼 문장이 끊어진다. PC SC/TC(그리고 PK EN)는 같은 세력 토큰 `025032`만 남기고 일본어 굴절 명령을 쓰지 않는다. 추가 다섯 사찰·신사 문구는 현재 PC `msgdata`의 활성 명칭(`주손지`, `릿샤쿠지`, `스와 대사`, `만간지`)과 대조했다.

## 수정 원칙

- PC JP/EN/SC/TC만 대조했다. Switch 한국어는 사용하지 않았다.
- 런타임 세력 토큰 `025032`은 바이트 그대로 보존한다.
- 각 레코드의 일본어 전용 `0143` 굴절 명령만 정확한 오프셋에서 제거한다.
- 수동 줄바꿈 수는 바꾸지 않는다. 모든 결과는 두 줄이다.
- 세력명이 길어질 때의 실제 UI 폭은 별도 실게임 QA 항목으로 남긴다.

## 기대 표시

`<세력명> 측과의 친선은`
`거리/비용/중개자 사유로 계속할 수 없습니다.`

빌더는 입력·출력의 전체 11개 텍스트 리소스 해시, 각 레코드의 리터럴·불투명 바이트·PC 원문·PC SC/TC 구조를 모두 고정한다. Steam에는 이 빌더가 만든 후보만 적용한다.
