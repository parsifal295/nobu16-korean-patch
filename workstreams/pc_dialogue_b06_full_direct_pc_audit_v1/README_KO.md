# PC MSGGAME B06 전수 직접 대조 감사 v1

## 범위와 원칙

- 대상은 `msggame.bin` block `06`뿐이다.
- 현재 Steam PC 한국어 파일과 지정된 pristine PC 일본어 파일을 **같은 record / literal 좌표**로만 직접 대조했다.
- Switch·다른 언어판·기존 B06 후보·기존 B06 보고서는 읽거나 근거로 쓰지 않았다.
- 이 workstream은 감사 보고서 하나만 만든다. 후보 `.bin`, Steam 적용, Git 저장소 명령, 커밋·푸시·릴리스·네트워크 작업은 수행하지 않았다.
- 아래의 전수 수치는 모든 B06 좌표를 파싱·대응시킨 구조/문자/정적 의미 감사 범위다. 런타임 토큰 결합, 인물별 말투의 문학적 완성도, 실제 화면 폭·줄바꿈 품질의 완료 선언은 아니다.

좌표는 `6:<record>:<literal>`로 표기한다. 텍스트가 아니라 제어 바이트를 추측해 고치는 일은 하지 않았다.

Base에 남아 있는 일본어 Unicode는 `6:4461:0-1`, `6:4462:0`, `6:4463:0`의 글머리표 `・` 4개뿐이다. 모두 `・임무`, `・건의` 목록의 bullet이며 일본어 어휘나 깨진 문자가 아니다.

사용자 보고와 직접 관련된 오탈자형 문자열 `덴령`, `상사람`, `소성`, `선교사`의 비정상 표기, `했습니다군`, `하였습니다군`, `나오했습니다군`은 이 B06 KO 범위에서 각각 0건이었다. 이 결과는 B06 범위에만 한정한다.

## 고신뢰 정적 수정 방향

원문의 뜻과 현재 한글이 직접 충돌하거나, 한글 문법·인칭이 명백히 무너진 항목만 분리했다. 처음 PK 22개를 독립 확인한 뒤, 같은 원문·같은 KO를 전 B06에서 다시 검색하여 혼인 문구의 PK 1개와 Base 5개를 추가로 찾았다. 따라서 고신뢰 대상은 **PK 23개 + Base 5개 = 28 literal**이다. 파일에는 아직 쓰지 않았다.

- 위 28개 대상 record는 모두 KO/JP literal 개수, record 단위 LF 수, 비리터럴 skeleton이 각각 일치한다. 즉 후보에서는 해당 literal만 바꾸고 제어 바이트·수동 LF를 그대로 보존할 수 있다.
- `6:1442:0`–`6:1453:0`의 두 번째 literal은 원문과 KO 모두 불완전한 결합 조각(`…당가의 중심이 되`)이므로 이번 정적 방향에서 건드리지 않는다.
- PK `6:2257:0`은 권장문이 원래의 한 줄짜리 2행보다 길다. **두 줄 구성 자체는 유지하되**, Steam 반영 전 실제 해당 대사 상자에서 폭·자동 줄바꿈을 확인해야 한다. 이 문서에는 화면 QA 결과가 없다.
- `6:4508:0`의 원문 끝은 활용 조각이므로, 권장 2행의 최종 어미는 후보 검증 시 실제 출력과 함께 확인한다. 방향은 청자 명령이 아닌 화자의 1인칭 의지형으로 고정한다.

## Hold

| 좌표 | KO / JP 차이 | Hold 사유 |
| --- | --- | --- |
| PK `6:1230:1`–`6:1232:1`, `6:1234:1`–`6:1238:1` | JP는 `援軍を頼める味方`(원군을 **청할 수 있는** 아군)인데 KO는 `원군을 **보낼** 아군`이다. 같은 JP의 PK `6:1233:1`은 이미 `원군을 청할 동맹국`으로 자연스럽다. | 의미 방향은 명백하지만 앞 literal에 동적 목표명이 결합된다. 원래 3행과 선행 이름 토큰이 실제 화면에서 어떻게 이어지는지, 폭이 유지되는지 확인한 뒤 literal 1만 `원군을 청할 아군을 늘리는 게 어떻겠습니까?` 방향으로 고친다. 정적 구조·LF·skeleton은 이미 일치한다. |

## 결론과 비변경 확인

- B06의 Base 6,635개와 PK 7,556개 literal 좌표는 pristine PC JP와 전부 대응시켜 구조·문자·정적 의미 신호를 점검했다.
- 후보로 검토할 고신뢰 방향은 PK 23개와 Base 5개, 합계 28 literal이며, 런타임/폭 검증이 필요한 Hold는 분리했다.
- 이 문서는 B06의 직접 PC 감사 기록이다. B06 또는 전체 프로젝트의 문학적 번역 품질이 완료되었다는 선언이 아니다.
- 생성물은 이 `README_KO.md` 하나이며, Steam 파일·후보 바이너리는 변경하지 않았고 Git 명령·커밋·푸시·릴리스는 수행하지 않았다.
