# Switch v1.3 화면 제목 이미지 카탈로그

Switch v1.3 `RES_JP/res_lang.bin`의 `/3/0..107`과 PC PK SC
`RES_SC/res_lang.bin`의 `/3/0..107`을 **구조적 슬롯 인덱스** 기준으로 대응시킨다.
이는 문구 의미까지 108개가 모두 일치한다는 뜻이 아니다. 공개 결과에는
ID, 해시, G1T/BC3 형식, 크기, 레이아웃 메타데이터와 사람이 확인한 한글 라벨만
들어간다.

## 픽셀·의미 전수 감사 결과

비공개 접촉 시트에서 108쌍을 전수 대조했다. Switch v1.3은 BC3/RGBA 기준
실제 고유 이미지가 각각 106장이고 완전 중복은 `[23, 42]`, `[37, 38]` 두 쌍이다.
PC 108장은 모두 고유하다. v1.1과 v1.3 사이에 같은 인덱스의 RGBA가 그대로인
항목은 0개다.

- PC 의미와 바로 번역 참고로 쓸 수 있는 항목: 101개
- placeholder: `/3/0` 1개. Switch 문구는 `임시 캡션`이고 PC 문구 의미는
  `거래 내용 파악`이므로 그대로 넣으면 안 된다.
- 확정 의미 순서 불일치: `/3/24`, `/3/25` 2개. `요리키 일람`과
  `요리코 선택`이 PC의 두 대응 슬롯과 반대로 배치되어 있다.
- 용어·구분 재검토 권장: `/3/15`, `/3/37`, `/3/38`, `/3/74` 4개.
  특히 37/38은 PC에서 서로 다른 두 조직 편제 문구인데 Switch 픽셀은 둘 다
  `부대 편제`로 완전히 같다.

따라서 **108장 전체를 인덱스 그대로 자동 이식하는 경로는 금지**한다. 감사 결과를
반영한 현재 의미 계획은 103개 같은 인덱스 재사용, 3개 재매핑, 2개 새 렌더로
108개를 모두 다룬다.

- 재매핑: `0 ← 3`, `24 ← 25`, `25 ← 24`
- 새 렌더: `38`의 `부대 편성`, `74`의 `공주 정보`
- 같은 인덱스 유지가 확정된 검토 항목: `15`의 `연조`, `37`의 `부대 편제`

Switch G1T/LINK 바이트는 복사하지 않는다. 각 번역 픽셀의 alpha bbox를 잘라 PC
원본 bbox 높이에 맞춰 등비 조절한 뒤, PC 512x128 캔버스의 왼쪽·위 잉크 원점에
배치하고 BC3/G1T/raw-LZ4/내부 LINK/외부 LINK를 다시 조립한다. 초기 2장 프로브를
넘어 108장 전체를 처리한 비공개 오프라인 PC 후보 A를 생성했으며 후보 SHA-256은
`057E3B7E1BA426EAAAC1ABE3A7907DB7E750BDAD9419E87743C5A5612D1A6E62`이다.
후보에서 108장을 모두 재추출했고 외부 LINK 비대상 엔트리와 PC 전용 꼬리 엔트리
`108`, `109`의 보존도 확인했다. 입력 아카이브와 게임 설치본은 수정하지 않았다.

후보 아카이브와 재추출 PNG는 완전한 게임 리소스 및 제3자 번역 픽셀을 포함하므로
`tmp/**/private/` 밖에 공개하거나 배포할 수 없다. 실제 게임 화면의 위치·크기·가독성
QA는 아직 남아 있으며, 그 검증 전에는 배포 판정이 아니다. 초기 전수 감사의 판단과
수량은 `catalog.v0.1.json`의 `semantic_pixel_audit`에 소스 비포함 메타데이터로
유지한다.

다음 데이터는 이 workstream에 넣지 않는다.

- Switch/PC `res_lang.bin`
- LINK/G1T/raw-LZ4/BC3 데이터
- 디코드한 PNG·PPM 또는 접촉 시트
- 서울한강체 글꼴 파일이나 게임 원본 이미지

렌더링 결과는 `tmp/` 아래에서만 검토한다. 판독이 확실한 항목만
`labels.review.json`에서 `status: verified`로 표시하고, 나머지는 카탈로그에서
`needs_review`로 유지한다.

```powershell
python -B workstreams\switch_title_images_v13_catalog\build_catalog.py `
  --switch-zip tmp\third_party_switch_v13\NobunagaShinsei_KoreanPatch_v1.3.zip `
  --pc-res-lang F:\Games\NOBU16\RES_SC\res_lang.bin
```

빌더는 Switch v1.3 ZIP, 내부 `RES_JP/res_lang.bin`, Switch `/3`, PC `/3`의
SHA-256을 고정 검증하고 108개 항목이 모두 `GT1G0600`, 단일 텍스처,
format `0x5B`(BC3/DXT5)인지 확인한다. 게임 설치 경로는 읽기 전용 입력이며 어떤
파일도 수정하지 않는다.
