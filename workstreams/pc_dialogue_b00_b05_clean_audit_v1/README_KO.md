# PC 인물 대사 감사: MSGGAME B00~B05 (읽기 전용)

## 결론

B00~B05의 Base/PK 한국어를 각각의 **pristine PC 일본어 원본**과 대조했다.
바로 수정 후보로 올릴 수 있는 고신뢰 용어 불일치는 **8개 literal**이다. 모두 일본어 `元服`을 `겐푸쿠`로 남긴 곳이며, 같은 PK B02의 동일 원문은 이미 `원복`으로 번역되어 있다.

이는 B00~B05의 전체 문학 품질이나 전체 번역 완료를 뜻하지 않는다. 정적 대조만으로 안전하게 판정할 수 없는 줄바꿈·레코드 envelope 차이는 아래 Hold로 남겼다. 후보 생성, Steam 적용, Git, 네트워크, 커밋, 릴리즈는 수행하지 않았다.

## 검사량과 topology

좌표 표기는 `블록:레코드:리터럴`이다. Base와 PK는 서로가 아니라 각자 대응하는 PC JP와 비교했다.

| 블록 | Base 레코드 쌍 | Base 리터럴 쌍 | PK 레코드 쌍 | PK 리터럴 쌍 |
| --- | ---: | ---: | ---: | ---: |
| B00 | 2,612 | 1,404 | 2,729 | 1,467 |
| B01 | 32 | 84 | 32 | 84 |
| B02 | 653 | 850 | 681 | 888 |
| B03 | 24 | 25 | 80 | 83 |
| B04 | 52 | 61 | 155 | 165 |
| B05 | 4 | 2 | 4 | 2 |
| 합계 | **3,377** | **2,426** | **3,681** | **2,689** |

- 레코드 ID/개수 topology 불일치: Base 0, PK 0
- 리터럴 개수·ID topology 불일치 레코드: Base 0, PK 0
- 색상 ESC, 런타임 대괄호 토큰, printf 토큰, 리터럴 내 비-LF 제어문자 signature 불일치: Base 0, PK 0
- 숫자 signature(전각 정규화 후) 불일치: Base 0, PK 0
- 한글 literal의 실제 일본 가나·CJK 잔재: Base 0, PK 0
- U+FFFD replacement character: Base 0, PK 0

B01의 `、` 84개와 Base B04의 `・` bullet 3개는 원본과 같은 delimiter/bullet glyph다. 일본어 문장 잔재로 분류하지 않았다.

## 고신뢰 오류

### `元服` 용어 일관성: `겐푸쿠` → `원복` (8 literal)

PK `2:94:1`, `2:95:1`은 같은 JP `が武将として元服`을 이미 `무장으로서 원복했습니다.`로 번역한다. 아래 8개는 동일 개념을 일본어 독음 `겐푸쿠`로 남겨 용어가 갈린다. 의미·태그·LF 구조는 유지하고, 프로젝트 내 기존 `원복` 용어로 통일하는 것이 안전한 수정 방향이다.

| 범위 | 좌표 | 현 KO | JP 원문 | 판정·수정 방향 |
| --- | --- | --- | --- | --- |
| Base | `2:88:1`, `2:89:1` | `이 무장으로 겐푸쿠` | `が武将として元服` | `겐푸쿠`를 `원복`으로 통일. 동적 앞부분 결합 문맥은 보존 |
| Base | `2:93:0` | `저도 겐푸쿠…아니, 성인식을 마쳤기에…` | `私も元服…いや、髪結の儀を終えましたゆえ…` | `겐푸쿠` → `원복`; 여성 성인식과 대조하는 문장 구조 유지 |
| Base | `2:105:0` | `겐푸쿠를 맞이해…` | `元服を迎え…` | `원복을 맞이해…` 방향 |
| Base | `2:106:0` | `겐푸쿠를 치르고…` | `元服して…` | `원복을 치르고…` 방향 |
| PK | `2:99:0` | `저도 겐푸쿠…아니, 성인식을 마쳤기에…` | `私も元服…いや、髪結の儀を終えましたゆえ…` | `겐푸쿠` → `원복` |
| PK | `2:111:0` | `겐푸쿠를 맞이해…` | `元服を迎え…` | `원복을 맞이해…` 방향 |
| PK | `2:112:0` | `겐푸쿠를 치르고…` | `元服して…` | `원복을 치르고…` 방향 |

아래 검사는 후보를 넓히기 위한 보조 필터로도 사용했다. 문맥상 false positive는 오류로 기록하지 않았다.

- 알려진 표기 재탐색: `덴령`, `소성씨`, `상사람`, `선교…`, `했습니다군`, `나왔습니다군`, `과분`, `역부족`은 이 범위에 없음.
- `伝令`은 PK `2:357:0`에서 `내 개를 전령으로 써서\n훌륭히 이끌어 보이겠다.`로 정상 표기됨.
- 성공/실패, 아군/적, 증감, 부정/가능성 반전 후보를 재검토했으며 Base `4:34:0`의 `개별 설정 가능` 한 건은 같은 문장 안의 별도 문맥으로 인한 false positive였다.

## Hold: 런타임·문맥 확인 필요

### 수동 LF 재배치

문장 의미가 뒤집혔다고 판단할 근거는 없으나, 동적 조각 결합 및 실제 UI 폭은 정적 JP 대조만으로 안전하게 확정할 수 없다. 따라서 수정하지 않았다.

| 범위 | 좌표 | KO LF / JP LF | 관찰 |
| --- | --- | ---: | --- |
| Base B02 | `2:246:0`, `2:246:1` | 1/0, 0/1 | `길 정비에는 자신이 있으니`와 후행 조각 사이의 LF 위치가 이동 |
| PK B02 | `2:152:1` ~ `2:157:1` | 각 1/0 | 방침 상태 문구의 후행 조각 앞에 LF가 추가됨 |
| PK B02 | `2:252:0`, `2:252:1` | 1/0, 0/1 | 길 정비 문구의 LF 위치가 이동 |
| PK B02 | `2:366:0`, `2:366:1` | 2/1, 0/1 | 동맹/공동 전투 문구의 LF 분배가 변경됨 |

### 리터럴 외부 envelope/skeleton 차이

literal text와 태그·런타임·제어 signature는 모두 대응하지만, B02 일부 레코드는 리터럴 밖의 raw envelope가 JP와 다르다. localized UTF-16 길이에 따라 offset 성분도 달라질 수 있어, 정적 hash 차이만으로 제어 코드 손상이라고 단정할 수 없다. 게임 런타임에서 해당 동적 메시지 조합을 확인하기 전에는 수정 금지 Hold다.

- Base B02: 19개 레코드
  `2:212, 2:216, 2:219, 2:224, 2:227, 2:235, 2:238, 2:243, 2:244, 2:246, 2:253, 2:408, 2:410, 2:412, 2:489, 2:502, 2:512, 2:519, 2:557`
- PK B02: 32개 레코드
  `2:218, 2:222, 2:225, 2:230, 2:233, 2:241, 2:244, 2:249, 2:250, 2:252, 2:259, 2:321, 2:325, 2:349, 2:357, 2:358, 2:359, 2:360, 2:361, 2:363, 2:364, 2:366, 2:415, 2:417, 2:419, 2:446, 2:447, 2:503, 2:516, 2:526, 2:533, 2:574`

### 용어 표기 확인 필요

PK `2:670:0`과 `2:671:0`의 JP `郡代`가 KO `군대`로 표기된다. 한자 독음으로는 가능하지만 `軍隊`(army)와 같은 표기여서 UI 의미가 오독될 수 있다. 현재 범위의 정적 JP 대조만으로 프로젝트의 직함 glossary를 확정할 수 없으므로 오류로 단정하거나 수정하지 않았다. 실제 화면의 직함 표기와 glossary를 확인한 뒤 `군대(郡代)` 등 명확화 여부를 결정할 Hold다.
