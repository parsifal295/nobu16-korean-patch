# PC 이벤트 품질 Wave 33 — 엄격 정적 문장 보정

현재 Steam PC 이벤트 테이블에서 PC JP/EN/SC/TC 대조로 확정할 수 있는
13개 Base/PK 문장쌍, 총 26셀만 private 후보로 보정한다.

- 모가미 요시아키 장면의 8개 문장쌍은 모친의 친오빠 관계를 `외숙부`로
  바로잡는다. 다른 장면의 `백부`·`숙부`를 일괄 치환하지 않는다.
- 존대 오탈자, 그림 속 인물 지시, 병사 손실 의미, 화살 분배 수식, 회담 종결
  표현을 각각 원문 문맥에 맞게 보정한다.
- 현재 Steam KO 셀 해시·목표 해시·Base/PK 일본어 동치·PC 다국어 셀 해시,
  ESC/런타임/printf 보존과 선언된 수동 줄 수를 검증한다. `MSGEV`(PK)는 활성
  이벤트 글꼴 기준 3줄·912px 제한도 검증한다.
- Switch 파일·Switch 한국어·과거 한국어 산출물은 읽지 않는다.
- 빌더는 `tmp/pc_event_quality_wave33_strict_static_v1/` 아래 private 후보만
  쓰며 Steam 적용·Git·네트워크·릴리즈 기능이 없다.

ID `5267`은 항의·호소·작은 충돌의 문체 선택에 대한 독립 검토 결론이 갈려 이번
엄격 배치에서 제외했다. 더 자연스러운 대안은 있을 수 있으나, 명백한 의미 오류로
확정하기 전에는 자동 반영하지 않는다.

Base ID `6772`, `6941`, `8776`, `8803`, `8947`, `9292`는 수동 줄바꿈이
바뀌므로, 글꼴 폭 검증과 별도로 실제 Base 이벤트 장면 QA가 필요하다.

Base `ev_strdata`의 `Text_message`는 PC UI 리소스상 448×100 logical px 슬롯으로
확인됐지만, 글리프 advance를 실제 렌더 폭으로 환산하는 비율·자동 줄바꿈·최대 줄 수는
정적으로 확인하지 못했다. 따라서 Base에 PK의 912px 또는 과거 1104px 임계값을
재사용하지 않으며, Base 폭은 기록만 하고 자동 통과 판정에는 쓰지 않는다.

이 후보는 이벤트 전수 감사 완료를 주장하지 않는다. 런타임 토큰, Base UI 폭,
실게임 트리거가 필요한 보류군은 계속 별도 검토 대상으로 남긴다.
