# 노부나가의 야망 신생 PK 한글 패치

Windows Steam판 `NOBU16PK.exe` 일본어(JP) 리소스 경로를 사용하는 비공식
팬메이드 한글 패치입니다. 대상은 Steam JP PK v1.1.7(`Update 1.1.7`, Steam
build `18823764`)이며, Steam/런처의 게임 언어도 반드시 **Japanese**로
선택해야 합니다.

현재 공개 안정판은 [v0.13.1](https://github.com/parsifal295/nobu16-korean-patch/releases/tag/v0.13.1)입니다.

배포 ZIP SHA-256: `7A670A03ACAB2CED43D8F27392CD1F31DB92CD16CB6B0EF04909B8E519A57FDF`
(380,397,094 bytes).

## 설치

1. 게임과 런처를 완전히 종료합니다.
2. ZIP의 모든 파일을 Steam 게임 폴더에 풀고 기존 리소스 덮어쓰기를 허용합니다.
   일반적인 경로는 `...\SteamLibrary\steamapps\common\NOBU16`입니다.
3. 게임 폴더의 `APPLY_STATIC_EXE_PATCHES.bat`를 한 번 실행합니다. 기존
   `APPLY_STATIC_OFFICER_EDITOR_FIX.bat`도 같은 마스터 설치기를 실행합니다.
4. 완료 후에는 평소와 같이 Steam → 1.1.7 런처 → 게임으로 시작합니다.

v0.11.1 이후의 정적 설치기는 게임 실행 중에 메모리를 건드리는 helper가 아닙니다. 사용자의
정확한 원본 EXE를 로컬에서 언팩·검증하고 정적 패치 EXE로 한 번 교체합니다.
매 게임 세션마다 별도 실행 파일을 실행할 필요가 없습니다.

## v0.11.1 — 설치기 경로·한글 오류 출력 수정

v0.11.0의 APPLY/RESTORE 배치 파일은 `%~dp0` 끝의 `\`를 PowerShell에 전달할
때 닫는 따옴표와 충돌해 `경로에 잘못된 문자가 있습니다.` 오류로 설치가
실패할 수 있었습니다. v0.11.1은 두 배치 파일 모두 정규화 가능한 `.` 접미
경로를 전달하도록 고쳤으므로, `E:\SteamLibrary\...\NOBU16`처럼 일반적인
Steam 라이브러리 경로에서도 적용·복구가 진행됩니다.

또한 설치 PowerShell 스크립트를 UTF-8 BOM으로 배포해 Windows PowerShell
5.1에서 `실패:` 같은 한글 오류 메시지가 깨져 보이던 문제를 수정했습니다.
게임 리소스 15개와 정적 EXE의 다섯 패치 지점은 v0.11.0과 동일합니다.

## v0.11.0 — 무장 에디트 한글 성명 저장

한글 `msgdata` 성명은 일반 화면/이벤트의 `msgev` 한글 표기와 별개로,
무장 에디트의 EXE 검증에 걸렸습니다. 그래서 기존 이름을 건드리지 않아도
`성명에 사용할 수 없는 문자가 포함되어 있습니다`가 나타났습니다.

이번 정적 패치는 다음 다섯 실패 분기를 고정된 원본 바이트 확인 뒤에만
수정합니다.

- 성·이름 표기 문자 검사 2곳
- 성·이름 읽기 문자 검사 2곳
- 성명 합계 6자 제한 1곳

따라서 일반 화면의 `msgev` 한글 이름은 그대로 유지하고, 무장 에디트에서는
`가가와 미초카게`처럼 한글 표기와 비가나 읽기 필드를 가진 기존 무장도
능력치 변경 후 저장할 수 있습니다. 테스트는 1920×1080 창 모드에서 게임을
완전히 종료·재실행한 뒤 Steam → 런처 → 게임 경로로 수행했고, 능력치 변경,
`결정`, 목록 복귀 및 저장을 확인했습니다.

### 이벤트 스크립트 전수 검수·보정

이번 배포본에는 이벤트 대사 표시용 `MSG_PK/JP/msgev.bin`의 전수 검수·보정
결과를 포함했습니다. 17,916개 문자열을 대상으로 4,003개 항목을 수동 검토하고,
한글 문맥에 맞춘 줄바꿈·표시 배치 3,774건을 보정했습니다. 대화 상자는 최대
3줄·줄당 912px 기준으로 재배치했으며, 동적 인명 토큰의 폭을 예약해 이름 길이
변화에 따른 넘침을 방지합니다.

설치기는 다음의 원본만 허용합니다.

```text
NOBU16PK.exe SHA-256
29BC1ED66D27B9AEF5EB6CE3D126BA2BDBF86099E12B09615FE9F988F41E2246
```

언팩 결과와 패치 전 5개 바이트열, 최종 SHA-256까지 일치해야 교체합니다.
최종 정적 EXE SHA-256은 다음과 같습니다.

```text
2E098ECB5E4335DC264F865306B990B724EA7C242B1B9F87FFC5EE2E7191797C
```

성공하면 게임 폴더에 `NOBU16PK.exe.staticfix.original_1.1.7` 원본 백업을
남깁니다. 원복하려면 게임과 런처를 종료한 뒤
`RESTORE_ORIGINAL_NOBU16PK_EXE.bat`를 실행하면 됩니다. Steam 파일 무결성
검사나 게임 업데이트도 EXE를 원본으로 되돌릴 수 있으며, 이 경우 정확히
같은 1.1.7 원본일 때만 다시 설치할 수 있습니다.

정적 패치 EXE는 Steam 보호 래퍼와 원본 Authenticode 서명이 제거된
`NotSigned` 파일입니다. 탐색기에서 EXE를 직접 실행하지 말고 Steam 또는
1.1.7 런처에서 시작하십시오. 설치 도구는 Windows .NET Framework 4.5.2
이상이 필요하며, 포함된 Steamless v3.1.0.5 구성요소는 atom0s의 원본 그대로
CC BY-NC-ND 4.0 조건으로 제공됩니다. 자세한 고지는 배포본의
`OfficerEditorStaticFix\THIRD_PARTY_NOTICES.txt`에 있습니다.

이 변경은 한글 IME 입력이나 새 한글 이름 작성 기능을 추가하지 않습니다.
무장 에디트에서는 성명·읽기 필드를 바꾸지 말고 능력치 등 다른 항목만
수정하는 용도를 권장합니다. 문자 종류와 합산 길이 검사는 이 검증기를 쓰는
해당 편집 화면에서 완화됩니다.

EXE를 바꾸지 않는 대안은 `msgdata`의 성명·읽기만 일본어 원문으로 되돌리는
호환 모드뿐입니다. 그 경우 일반 화면의 `msgev` 한글은 유지할 수 있지만,
편집·선택 화면의 장수명은 일본어로 표시됩니다. 한글 표기를 유지한 채
통과시키는 순수 리소스 해법은 없습니다.

보장하지 않습니다.

## 글꼴과 권리

한글 글리프는 서울한강체 계열을 사용하며, 32px/4px 계열은 서울한강 B,
48px/6px 계열은 서울한강 EB로 렌더링했습니다.

이 프로젝트는 비공식 팬메이드 한글 패치입니다. KOEI TECMO GAMES의 공식
제품·지원물이 아니며, 게임·상표·로고·원본 데이터의 모든 권리는 KOEI
TECMO GAMES 및 각 권리자에게 있습니다. `NOBU16PK.exe` 게임 실행 파일은
배포 ZIP에 포함하지 않습니다.
