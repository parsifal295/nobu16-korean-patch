NOBU16 PK Steam JP 1.1.7 v0.14.0
레지스트리 기반 정적 EXE 패치 설치기

이 ZIP에는 NOBU16PK.exe가 들어 있지 않습니다. 설치기는 사용자의 Steam JP 1.1.7
원본 EXE를 PC 안에서 한 번 언팩하고, 등록된 패치별 상태를 검사한 뒤 Pending인
항목만 적용합니다. 이미 적용된 항목은 건너뜁니다. 런타임 메모리 패처는 없습니다.

등록된 패치
- 001: 무장 편집기 한글 성명 검증
- 002: 가공 히메 한글 작명 검증
- 003: 한국어 상단 헤더 가로 배치
- 004: 지도 성·지명 가로쓰기와 문자열 길이 기반 동적 배경 라벨
- 005: 고해상도 지도 라벨 아틀라스와 명소 이름 가로쓰기
- 006: 지도 성 이름 뒤 상태 아이콘 동적 정렬
- 007: 이벤트 메시지 글자 크기와 줄간격 조정

004는 v0.12.0에서 배포한 정의와 해시를 그대로 유지합니다. 005는 4K에서
저해상도 지도 라벨 아틀라스가 선택되어 그래픽이 깨지던 문제와, 아쓰타 신궁 같은
명소 이름이 세로쓰기 컨테이너 안에서 한 점에 겹치던 문제를 함께 수정합니다.
006은 성 이름 라벨과 별도로 갱신되던 전투 준비·방위 거점·공략 목표 아이콘을
실제 라벨 폭 뒤에 배치합니다. 상태 아이콘의 세로 위치도 현재 라벨과 아이콘
높이에서 계산해 라벨 중앙에 맞춥니다. 고정 X/Y 보정값은 사용하지 않습니다.
성 문장 아이콘과 그 바로 아래의 병사수(병종 아이콘+숫자) 표시 그룹, 특수 지도
커서 경로는 변경하지 않습니다.
007은 이벤트 메시지 전용 생성 루틴의 글자 크기를 36에서 30으로, 줄간격을
10에서 8로 조정합니다. 일반 UI, 무장 목록, 지도 라벨 등 다른 텍스트 위젯의
글꼴 설정은 변경하지 않습니다.

업그레이드 동작
- 중요: v0.13.1 사용자는 v0.14.0 파일을 게임 폴더에 덮어쓴 뒤 반드시
  새 APPLY_STATIC_EXE_PATCHES.bat를 다시 실행해야 합니다. 파일만 덮어쓰면
  EXE의 007은 적용되지 않습니다.
- v0.13.1에서 001~006이 적용된 EXE: 007만 자동 적용
- v0.13.0에서 001~005가 적용된 EXE: 006과 007을 순서대로 자동 적용
- v0.12.0에서 001~004가 적용된 EXE: 005~007을 순서대로 자동 적용
- 원본 Steam JP 1.1.7 EXE: 001~007을 순서대로 적용
- v0.14.0 재실행: 모든 항목을 Applied로 판정하고 EXE를 변경하지 않음

지원 대상
- Steam 신장의 야망·신생 with 파워업키트 일본어판 Update 1.1.7
- 원본 NOBU16PK.exe SHA-256:
  29BC1ED66D27B9AEF5EB6CE3D126BA2BDBF86099E12B09615FE9F988F41E2246

설치
1. 게임과 런처를 완전히 종료합니다.
2. ZIP의 모든 파일을 게임 폴더에 덮어씁니다.
3. APPLY_STATIC_EXE_PATCHES.bat를 실행합니다.
4. Completed가 표시되면 평소처럼 Steam에서 실행합니다.

상태 판정 및 안전장치
- 각 항목을 Applied, Pending 또는 선행 구조 패치 대기 상태로 판정합니다.
- 004와 005의 연속 AppendOverlay 뒤에 006과 007 BytePatch를 등록 순서대로 적용하고
  역순으로 정규화합니다.
- 일부 바이트만 적용된 상태, 알 수 없는 EXE, 변조된 추가 페이로드는 거부합니다.
- 설치 전 원본은 NOBU16PK.exe.staticfix.original_1.1.7로 보관합니다.
- 모든 항목 적용 후 EXE SHA-256:
  F424964405CFCD1AC454B3801DA4795A183A8271DD16EA8A6A7B97A2547232BF

실게임 및 설치기 확인
- 명소 이름 QA: 800x450, NOBU16PK.exe 완전 종료·재실행 후 아쓰타 신궁 가로 표시 확인
- Issue #72 지도 아이콘 QA: 1920x1080, NOBU16PK.exe 완전 종료·재실행 후
  전투 준비·방위 거점·공략 목표는 동적 성 이름 끝에 정렬되고 병사수 표시는
  성 문장 바로 아래의 원래 위치를 유지함을 확인
- 이벤트 메시지 QA: 1920x1080, NOBU16PK.exe 완전 종료·재실행 상태에서
  글자 크기 30과 줄간격 8을 적용하고 다음 메시지 페이지에서도 유지됨을 확인
- v0.13.1 설치본에서는 007만 적용되는 업그레이드 경로 확인
- 원본 Steam EXE에서는 001~007 전체 신규 설치 확인
- 같은 설치기를 다시 실행하면 모든 항목이 Applied로 유지되는지 확인

복원
- 게임과 런처를 종료한 뒤 RESTORE_ORIGINAL_NOBU16PK_EXE.bat를 실행합니다.
- Steam 파일 무결성 검사나 게임 업데이트가 EXE를 원본으로 되돌릴 수 있습니다.

제한
- Steam JP 1.1.7의 정확한 원본 또는 이 설치기가 만든 등록 상태만 지원합니다.
- 한국어 IME 입력 기능 자체를 추가하는 패치가 아닙니다.
- 패치 후 EXE는 원본 Authenticode 서명이 제거된 NotSigned 파일입니다.
- Windows의 .NET Framework 4.5.2 이상이 필요합니다.
